    Visible modules

    This is not a standalone kiosk. It is a lightweight operating layer for the whole waiting-room experience.

    Each module has a clear role, but all of them share the same state so patient, reception, and leadership work from the same truth.

    Kiosk

    Guided in-room check-in

    For scheduled and walk-in patients, with quick help, large text mode, and one-tap receptionist support.
    • Scheduled vs walk-in split
    • Senior-first flow with voice guide
    • Thermal printing and offline safety

    Waiting room display

    Privacy-aware live calling

    The screen shows ticket, initials, and consult room so the room understands what is happening without exposing sensitive data.
    • Configurable bell
    • Current call and next tickets
    • Local snapshot for continuity

    Reception / Admin

    Queue operations for real consult rooms

    Call, reprint, complete, flag no-show, and control stations from a console built for active front desk teams.
    • Station or consult room lock
    • Numpad shortcuts and bulk actions
    • Traceable sync state and activity

    FAQ

    What clinics ask before buying

    Do we need proprietary hardware to launch?

    No. The pilot is designed to work with a browser, a TV or monitor, a PC or tablet, and a compatible thermal printer.

    Does this replace the clinic's full medical software?

    Not at this stage. The suite competes as waiting room operations + booking + reminders + reporting, not as a full EMR.

    Can we start with a single site?

    Yes. The first model is built for a pilot site, then expands to branding, lite multi-site, and dashboard layers only if the flow proves itself.

    How do you reduce no-show if patients never touch the kiosk?

    The improvement does not depend on the kiosk alone. It comes from reminders, mobile check-in, queue visibility, and a front desk that sees the same state.

    Walk the suite in the same order a buyer would

    First the promise is clear. Then the check-in becomes real. Then queue status lowers uncertainty. Finally the dashboard justifies budget and renewal.

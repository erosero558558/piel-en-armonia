### TAREA-M `[codex_backend]` — DEBT-03: Migrar write_store() restantes a mutate_store() (60 min)

**¿Por qué importa?** Las race conditions con write_store() directo pueden corromper el store con 2+ médicos simultáneos. Esto es corrupción de datos de producción.

**Diagnóstico actual:**
```bash
grep -rn "write_store(" controllers/ | grep -v "mutate_store\|with_store_lock\|#\|//" | wc -l
# → El resultado es el número de casos críticos
```

**Prioridad de migración** (hacerlo en este orden):
1. `TelemedicineRoomController.php` — buscar `write_store(` y envolver en `mutate_store()`
2. `ReviewController.php` — mismo patrón
3. Cualquier otro controlador con >1 write_store() no protegido

**Patrón de migración:**
```php
// ANTES (inseguro):
$store = read_store();
$store['key'] = $newValue;
write_store($store);

// DESPUÉS (atómico):
mutate_store(function(array $store) use ($newValue): array {
    $store['key'] = $newValue;
    return ['ok' => true, 'store' => $store, 'storeDirty' => true];
});
```

**Criterio de éxito:**
```bash
grep -rn "write_store(" controllers/ | grep -v "mutate_store\|with_store_lock\|//\|#" | wc -l
# → resultado < 35 (actualmente ~45, meta bajar al menos 10)
```

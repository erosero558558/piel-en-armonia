# AG-021 - Tech debt P3: automate C1 exit gates in CI

## Contexto

- Task: `AG-021`
- Objetivo: agregar un gate C1 en CI que ejecute contrato Figo y smoke en staging antes de `build`/merge.

## Cambios aplicados

Archivo modificado:

- `.github/workflows/ci.yml`

Implementacion:

1. Nuevo job `c1-exit-gate` (bloqueante para `build`).
2. `needs: [lint]` y `timeout-minutes: 25`.
3. Usa `TEST_BASE_URL` desde `vars.STAGING_URL` (fallback `https://staging.pielarmonia.com`).
4. Resuelve URL efectiva para evitar falso rojo por DNS:
    - intenta staging
    - si staging no resuelve, usa `vars.PROD_URL` / `https://pielarmonia.com`
    - si ninguna resuelve, falla explĂ­citamente
5. Ejecuta solo cuando hay cambios relevantes (paths-filter `c1`).
6. Ejecuta:
    - `npm ci`
    - `npx playwright install --with-deps chromium`
    - `npm run test:figo-contract` contra staging
    - smoke de endpoints:
        - `GET /api.php?resource=health` (2xx + payload valido)
        - `GET /api/chat` (error solo si 5xx; 404 queda warning controlado)
        - `GET /figo-chat.php` (2xx + payload valido)
7. `build` ahora depende de `c1-exit-gate`:
    - `needs: [lint, security, unit-tests, e2e-tests, c1-exit-gate]`

# Public V6 Image Refresh

Update as of March 12, 2026: the public V6 runtime no longer uses the
editorial SVG illustration library as its canonical source. The site now
publishes a photo-first asset family generated from approved raster masters in
`images/src/**`.

## Scope

- Public brand site only: `/es`, `/en`, home, hub, service, telemedicine, legal, shared SEO/preloads, and legacy public fallbacks that still resolve image paths.
- Excluded: admin, kiosk, operational apps, logos, PWA icons, archived `public-v3|v4|v5`, and verification artifacts.
- Guardrail respected: no source edits in `src/apps/astro/src/components/public-v6/SoftwareSurfacePageV6.astro` or `src/apps/astro/src/styles/public-v6/components.css`.

## Production Notes

- Asset family prefix: `v6-clinic-*`.
- Master source format: `images/src/*.jpg|*.png`.
- Derived runtime assets: `images/optimized/*.jpg`, `*.webp`, and `*-lqip.jpg`.
- Breakpoints generated by the local pipeline:
    - `wide`: `400, 640, 800, 1024, 1200, 1344, 1400`
    - `card`: `400, 640, 800, 900, 1200, 1400`
    - `portrait`: `500, 900`
- Unsuffixed `.jpg` and `.webp` point to the largest generated variant.
- No external image-generation credentials are required at build time. Any AI-assisted scene must be approved before entering the repo as a raster master. The repo only crops, composes, and derives runtime artifacts through `src/apps/astro/scripts/generate-public-v6-image-set.mjs`.

## Prompt Pack

### Base Art Direction

`premium dermatology photography, calm consultation energy, real skin and real clinic surfaces, restrained editorial framing, soft medical lighting, natural texture, high legibility, no logos, no text baked into the art`

### Shot Rules

- Keep every scene human-centered and medically calm, not cartoon or flat-illustrated.
- Doctors and team surfaces must use approved real staff photography.
- Generic clinical scenes may use real photography or approved AI-photoreal masters already committed into `images/src/**`.
- Reserve wide scenes for brand/SEO heroes, square or vertical scenes for service and team usage.
- Match each service to a distinct photoreal crop or composition even when the same source library is reused.

## Canonical Runtime Mapping

- `content/public-v6/es/*.json` and `content/public-v6/en/*.json` point to the `v6-clinic-*` family with reuse capped by validation.
- `PublicShellV6.astro` and route surfaces advertise route-specific `og:image` values from the visible hero.
- Legacy public fallbacks only resolve the `v6-clinic-*` family; no `showcase-*`, `service-*`, `team-*`, or `hero-woman` assets remain active on public routes.

## Generator Contract

- Script: `src/apps/astro/scripts/generate-public-v6-image-set.mjs`
- Input: approved raster masters from `images/src/**`, plus a local composition map and localized alt copy
- Output:
    - one JPG master per asset in `images/src`
    - responsive JPG/WebP variants in `images/optimized`
    - one `-lqip.jpg` per asset
- Existing V6 SVG masters are removed from the canonical path as each photo asset is regenerated.
- Rasterization is performed locally through Playwright to keep the pipeline reproducible inside the repo.

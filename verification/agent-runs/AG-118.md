# AG-118 - Numpad por estacion C1/C2 en admin queue

## Objetivo
Implementar llamado operativo por numpad en dos PCs con estacion fija por URL de provision, evitando errores de consultorio y sin cambios backend/API.

## Cambios implementados
- `src/apps/admin/index.js`
  - Configuracion persistente de estacion en `localStorage`:
    - `queueStationMode` (`locked|free`)
    - `queueStationConsultorio` (`1|2`)
  - Bootstrap por URL:
    - `?station=c1|c2&lock=1|0`
    - persiste configuracion y limpia query params con `history.replaceState`.
  - Atajos numpad en contexto de turnero:
    - `NumpadEnter` llama siguiente del consultorio de la estacion en modo lock.
    - `Numpad1/Numpad2` solo cambian consultorio en modo libre.
    - `Enter` normal no llama.
    - Guardrails: ignora modificadores, `repeat`, y foco en campos editables.
  - Guardrail operativo en lock:
    - bloquea llamadas/cambios hacia consultorio opuesto.
    - toasts operativos de bloqueo/llamado.
  - Observabilidad frontend:
    - `CustomEvent('piel:queue-ops')` con eventos:
      - `station_bootstrap`
      - `station_locked_call`
      - `station_change_blocked`
      - `station_config_updated`
- `admin.html`
  - Nuevo panel `Control de estacion` dentro de `#queue`:
    - badges de modo y estacion.
    - botones de lock C1/C2, modo libre, reconfiguracion.
- `admin.css`
  - Estilos del panel de estacion, estados activos y accesibilidad visual.
  - Estado bloqueado para acciones del consultorio opuesto.
- `src/apps/admin/modules/queue.js`
  - Ajuste de copy de atajos operativos para incluir numpad.
  - Hardening de sincronizacion:
    - fallback automatico a `queue-state` cuando `/api.php?resource=data` falla.
    - fallback automatico a `queue-state` cuando `/data` responde `ok` pero sin `queue_tickets`.
  - Mensaje de tabla vacia mas preciso cuando hay filtro/busqueda activa.
- `src/apps/admin/index.js`
  - Ajuste de lock por estacion:
    - mantiene guardrails para numpad/shortcuts.
    - permite override manual por boton para consultorio opuesto con toast y telemetria.
    - no fuerza `disabled=false` sobre botones ocupados para respetar estado real de C1/C2.
- `tests/admin-queue.spec.js`
  - Escenario `station=c2&lock=1`: valida `NumpadEnter` -> C2, `Enter` normal sin llamado, persistencia tras reload.
  - Escenario `station=c1&lock=1`: valida bloqueo de cambio por numpad y no llamado con foco en `#queueSearchInput`.
  - Escenario de override manual en lock para consultorio opuesto.
  - Escenario de fallback cuando `/data` no incluye `queue_tickets`.
- `docs/RUNBOOK_TURNERO_SALA.md`
  - URLs de provision por estacion y recovery rapido tras limpieza de cache.
  - Regla operativa explicita de override manual en lock (sin afectar atajos numpad).

## Validacion ejecutada
- `npm run lint:js` (sin errores; warnings preexistentes fuera de este bloque).
- `npx playwright test tests/admin-queue.spec.js` (`14 passed`).
- `npm run build` (bundle admin actualizado).
- `node agent-orchestrator.js board doctor --json` (`ok: true`).

## Resultado operativo
- PC provisionada con `admin.html?station=c1&lock=1` llama siempre C1 con `NumpadEnter`.
- PC provisionada con `admin.html?station=c2&lock=1` llama siempre C2 con `NumpadEnter`.
- Enter del teclado principal no dispara llamados.
- Configuracion persiste por navegador/PC y se recupera por URL en menos de 1 minuto.

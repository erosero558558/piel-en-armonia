# S7-05 — permisos por rol en OpenClaw

## Resumen

- Se agrego resolucion de rol clinico en `lib/auth.php` para distinguir `doctor` y `receptionist`.
- Se definieron scopes OpenClaw:
  - `assistant` -> `doctor|receptionist`
  - `clinical_write|prescription|certificate` -> `doctor`
- `OpenclawController` ahora usa scopes explicitos por endpoint y protege tambien `GET openclaw-prescription`.
- Se expusieron capacidades derivadas (`openclawDoctor`, `openclawReceptionist`, `openclawPrescription`, `openclawCertificate`) en el payload de auth.
- Se documento la configuracion en `env.example.php` y `docs/SECURITY.md`.

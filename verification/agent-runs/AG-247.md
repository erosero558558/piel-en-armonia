# AG-247 - Focus trim: required_checks runtime por surface

## Resumen

Se ajusto la evaluacion de `focus_required_checks` para aceptar checks runtime por
surface, no solo por provider completo. El foco activo del corte admin ahora usa
`runtime:operator_auth` en lugar de heredar el rojo global de
`runtime:openclaw_chatgpt`.

## Cambios

- `tools/agent-orchestrator/domain/focus.js`
    - se agrego resolucion de runtime requerido por surface
    - se mantiene compatibilidad con checks provider-wide
    - `buildFocusSeed()` pasa a sembrar `runtime:operator_auth` para el corte
      `FOCUS-2026-03-admin-operativo-cut-1`

## Validacion

```bash
node agent-orchestrator.js focus status --json
node agent-orchestrator.js runtime verify openclaw_chatgpt --json
npm run agent:policy:lint
npm run agent:test
```

## Resultado observado

- `focus status --json`
    - `required_checks: job:public_main_sync=unverified, runtime:operator_auth=green`
    - el foco deja de quedar bloqueado por surfaces runtime no relevantes al corte
- `runtime verify openclaw_chatgpt --json`
    - provider global sigue `unhealthy`
    - `figo_queue=degraded(legacy_proxy_without_gateway)`
    - `leadops_worker=unhealthy(worker_disabled)`
    - `operator_auth=healthy`
- `npm run agent:policy:lint`
    - OK
- `npm run agent:test`
    - `506/506` OK

## Decision operativa

El corte admin mantiene la exigencia runtime correcta para su prueba comun:
`operator_auth` debe estar sano; `figo_queue` y `leadops_worker` siguen siendo
deuda real del provider, pero ya no contaminan este foco especifico.

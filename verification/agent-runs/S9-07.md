# S9-07 — Origen de lead consistente

## Resumen

- Se agregó una ruta canónica en `LeadOpsService` para normalizar y persistir `source`, `campaign`, `surface` y `service_intent`.
- `normalize_callback()` y `normalize_appointment()` ahora graban esos cuatro campos en el record persistido.
- Los ingests de preconsulta, telemedicina y WhatsApp alimentan el origen comercial con defaults consistentes.
- El frontend de booking agrega `campaign` desde query string y conserva `surface/service_intent` al construir el payload.

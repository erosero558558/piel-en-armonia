# CDX-044 - Turnero Sony Polish 2 con foco en modo dia

Estado: `review`
Inicio: `2026-03-14`
Actualizado: `2026-03-14`
Estrategia: `STRAT-2026-03-turnero-web-pilot`
Subfrente: `SF-frontend-turnero-web-pilot`

## Objetivo

- Hacer una pasada global de pulido sobre `admin`, `operador`, `kiosco` y `sala`.
- Corregir el modo dia para que deje de verse como una aclaracion del oscuro y pase a sentirse como un sistema Sony showroom claro.
- Mantener contratos funcionales, IDs, hooks de Playwright y comportamiento operativo.

## Archivos trabajados

- `ops-design-system.css`
- `queue-ops.css`
- `queue-kiosk.css`
- `queue-display.css`
- `tests/turnero-sony-premium-contract.spec.js`

## Implementacion

- Se replanteo la paleta clara compartida en `ops-design-system.css` con blancos minerales, grises frios, acero suave y sombras mas limpias para las familias `command` y `ambient`.
- En `queue-ops.css` se rehizo la lectura diurna de `admin` y `operador`: mejor jerarquia entre hero, bandas, stream y CTA, menos fragmentacion visual, mejores pills/badges y tablas menos planas.
- En `queue-kiosk.css` el claro paso a una composicion mas guiada: hero de check-in mas fuerte, formularios mejor agrupados, tarjeta vacia de ticket intencional y lateral menos utilitario.
- En `queue-display.css` la sala clara se llevo a un look broadcast mas premium: hero mas dominante, reloj/estado encapsulados, cards de llamado con mas presencia y setup menos lavado.
- El contrato `tests/turnero-sony-premium-contract.spec.js` se endurecio para que el `150/150` tambien mida materialidad del modo dia, profundidad visual y mejor uso del espacio, no solo claro/oscuro basico.

## Resultado visible validado

- `admin` en claro se ve mas denso, mas sobrio y menos largo/fragil visualmente.
- `operador` en claro gana mejor shell operativo y CTA principal mas claro.
- `kiosco` en claro ya no depende del oscuro para sentirse premium; el espacio vacio del ticket pendiente queda tratado como parte del layout.
- `sala` en claro ya no parece una version desaturada: el bloque de llamada y la cabecera tienen mas presencia.
- El modo dia sigue automatico via `system` y no se agrego toggle visible.

## Comandos ejecutados

- `npx playwright test tests/turnero-sony-premium-contract.spec.js --workers=1`
- `npx playwright test tests/admin.spec.js -g tema --workers=1`
- `TEST_LOCAL_SERVER_PORT=8034 npm run test:turnero:web-pilot:ui`

## Resultado

- `tests/turnero-sony-premium-contract.spec.js` vuelve a pasar en verde con `150/150` despues del pulido.
- `tests/admin.spec.js -g tema` pasa `3/3`, confirmando que el comportamiento de tema del admin fuera de queue sigue sano.
- `npm run test:turnero:web-pilot:ui` pasa `139/139` en `8034`; hubo un intento previo fallido solo por puerto local ocupado (`8011`), sin regresion funcional del producto.
- La entrega queda lista para review visual con nueva evidencia de dia y noche para las 4 superficies.

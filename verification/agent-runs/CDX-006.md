## Objetivo

Extraer configuracion, repositorio, gateway y procesamiento de `lib/figo_queue.php`, y separar creacion/prioridad/resumen de tickets fuera de `lib/QueueService.php`, manteniendo el contrato procedural existente para chat, polling y cola.

## Cambios implementados

### Cola de turnos

- Nuevo modulo [lib/queue/TicketFactory.php](../../lib/queue/TicketFactory.php)
- Nuevo modulo [lib/queue/TicketPriorityPolicy.php](../../lib/queue/TicketPriorityPolicy.php)
- Nuevo modulo [lib/queue/QueueSummaryBuilder.php](../../lib/queue/QueueSummaryBuilder.php)
- [lib/QueueService.php](../../lib/QueueService.php) conserva su API publica, pero ahora delega:
    - creacion de tickets a `TicketFactory`
    - prioridad/orden a `TicketPriorityPolicy`
    - resumen/estado admin a `QueueSummaryBuilder`

### OpenClaw queue / Figo queue

- Nuevo modulo [lib/figo_queue/QueueConfig.php](../../lib/figo_queue/QueueConfig.php)
- Nuevo modulo [lib/figo_queue/JobRepository.php](../../lib/figo_queue/JobRepository.php)
- Nuevo modulo [lib/figo_queue/GatewayClient.php](../../lib/figo_queue/GatewayClient.php)
- Nuevo modulo [lib/figo_queue/JobProcessor.php](../../lib/figo_queue/JobProcessor.php)
- [lib/figo_queue.php](../../lib/figo_queue.php) queda como facade procedural estable y solo enruta a esas clases.

### Cobertura nueva

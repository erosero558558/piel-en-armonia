# CDX-060 - Admin queue pilot UI specs and fixtures

## Resumen

- Se reemplazo `tests/admin-queue.spec.js` por cinco specs focalizadas:
  - `tests/admin-queue-guidance-live-ops.spec.js`
  - `tests/admin-queue-reception-escalation.spec.js`
  - `tests/admin-queue-controls-numpad.spec.js`
  - `tests/admin-queue-pilot-governance.spec.js`
  - `tests/admin-queue-ops-hub.spec.js`
- Se agrego `tests/helpers/admin-queue-fixtures.js` para compartir helpers del piloto `admin-queue`.
- La adaptacion al trim visual del piloto abre explicitamente los grupos `details` de validacion y consolas avanzadas cuando el spec necesita inspeccionarlos.
- Los mount hosts del ops hub que hoy funcionan como wrappers se validan con `toBeAttached()` mientras que los componentes montados visibles siguen validandose con `toBeVisible()`.
- El runtime local del clone actualizado requirio una regeneracion previa con `npm run build:turnero:runtime` porque `admin.js` apuntaba a `js/admin-chunks/index-BFFMNQf-.js` y el chunk no estaba presente tras checkout limpio.

## Comandos ejecutados

```bash
npm run build:turnero:runtime
TEST_LOCAL_SERVER_PORT=9164 TEST_REUSE_EXISTING_SERVER=0 npx playwright test tests/admin-queue-guidance-live-ops.spec.js tests/admin-queue-reception-escalation.spec.js tests/admin-queue-controls-numpad.spec.js tests/admin-queue-pilot-governance.spec.js tests/admin-queue-ops-hub.spec.js tests/queue-integrated-flow.spec.js --workers=1
```

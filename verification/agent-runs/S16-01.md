# S16-01 — Psalm bootstrap critico

## Resumen

- Se instaló `vimeo/psalm` como dependencia de desarrollo en `composer.json` y `composer.lock`.
- Se ajustó `psalm.xml` para usar baseline explícita, flags estables (`findUnused*`) y un autoloader dedicado en `psalm-autoload.php`.
- Se creó `psalm-baseline.xml` a partir del barrido real del scope bootstrap.
- Se separó un job `psalm` en `.github/workflows/ci.yml` con `continue-on-error: true` para visibilidad inicial independiente de `security`.

## Bootstrap carve-outs

Para que el bootstrap inicial sea tractable en esta pasada, `psalm.xml` deja fuera temporalmente:

- `lib/clinical_history/ClinicalHistorySessionService.php`
- `lib/LeadOpsService.php`
- `controllers/OpenclawController.php`

Estos tres archivos mostraron timeouts patológicos (>25s por archivo aislado) y requieren tuning específico en una siguiente tarea antes de sumarlos al baseline.

## Validación

- `./vendor/bin/psalm --ignore-baseline --no-cache --set-baseline=psalm-baseline.xml`
- `./vendor/bin/psalm --no-cache`
- `git diff --check`

## Resultado

- `psalm-baseline.xml` quedó generado.
- `./vendor/bin/psalm --no-cache` cerró con `No errors found!` usando baseline.
- El barrido bootstrap reportó ~`91.96%` de inferencia tipada sobre el scope analizado.

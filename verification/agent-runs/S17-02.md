# S17-02 — Redencion de gift card en booking y cierre clinico

## Resumen

- Se agrego validacion en tiempo real de gift cards al formulario canonico de booking, persistiendo el contexto de la gift card en la cita antes del checkout.
- Se implemento `GET /api.php?resource=gift-card-validate` para consulta publica de saldo disponible y `POST /api.php?resource=gift-card-redeem` para redencion autenticada desde admin con actualizacion atomica de la cita.
- Se conecto el cierre clinico en admin para aplicar la gift card de la cita activa, refrescar la HCE y reflejar el estado financiero del encuentro.
- Se endurecio la cobertura de integracion para que el runner de GiftCardController levante sesion admin de prueba con CSRF por subproceso, evitando falsos 401 y sin serializar la prueba concurrente por lock de sesion.

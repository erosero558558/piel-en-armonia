# AG-190 - Turnero release ops stabilization

Fecha: 2026-03-11
Owner: Ernesto
Executor: codex

## Objetivo

Cerrar la estabilizacion operativa del release nativo del turnero sin romper el contrato canonico del registry, del bundle publicable ni de las superficies web asociadas.

## Cambios consolidados

- Se mantuvo `data/turnero-surfaces.json` como fuente unica para `operator`, `kiosk` y `sala_tv`.
- El runbook de release ya referencia el registry canonico, el scaffold y el dry run del bundle.
- Se agrego un gate unico del frente turnero en `package.json` para contratos Node + E2E Playwright.
- Se alineo el contrato de readiness del operador entre la UI actual y `tests/queue-operator.spec.js`.

## Resultado

- Gate canonico del turnero en verde: `14` contratos Node y `53` pruebas UI.
- Dry run sintetico del bundle genero `app-downloads/stable/release-manifest.json`, `app-downloads/stable/SHA256SUMS.txt` y feeds `desktop-updates/stable/...`.
- El manifest del dry run resolvio rutas publicas reales para `operator`, `kiosk` y `sala_tv`.
- Sin conflictos bloqueantes de board; los warnings restantes son operativos y externos a este frente (`public_main_sync` y WIP ajeno).
- La validacion PHP formal sigue pendiente de reparar: `vendor/bin/phpunit` continua roto por autoload en este workspace.

# AG-252 - CLI blocked_reason support para tareas bloqueadas

## Resumen

Se cerro la brecha entre el board y el CLI: ahora `task claim` y `task start`
pueden mover una tarea a `blocked` con `blocked_reason` explicito, y fallan si
se intenta bloquear una tarea sin esa razon.

Esto permite reflejar bloqueos reales sin editar `AGENT_BOARD.yaml` a mano y
sin dejar warnings evitables en `board doctor`.

## Cambios

- `tools/agent-orchestrator/commands/task.js`
    - agrega `--blocked-reason` al uso del comando
    - aplica `blocked_reason` cuando `status=blocked`
    - limpia `blocked_reason` al salir de `blocked`
    - rechaza `--blocked-reason` fuera de `status=blocked`
    - exige `blocked_reason` no vacio para tareas bloqueadas
- `tests-node/agent-orchestrator-cli.test.js`
    - cobertura para `task claim/start` con `blocked_reason`
    - cobertura para error al bloquear sin contexto
    - cobertura para limpieza del `blocked_reason` al reactivar la tarea

## Validacion

```bash
node --test tests-node/agent-orchestrator-cli.test.js
npm run agent:test
```

## Resultado

El CLI ya puede expresar bloqueos verdaderos de forma gobernada, que era el

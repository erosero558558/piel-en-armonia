# AG-255

## Alcance

- Reorden de la portada de `admin.html` para OpenClaw con foco operativo.
- Separacion visible entre via activa, estado del acceso y siguiente paso.
- Contingencia `clave + 2FA` solo cuando el backend la habilita.
- Ajuste de copy y estados para `local_helper`, `web_broker` y contingencia.

## Validacion local

- `npx playwright test tests/admin-openclaw-login.spec.js`
- `npm run test:admin:openclaw-auth`
- `npm run test:frontend:qa:admin`
- `npm run build`

## Resultado

- Todas las validaciones locales quedaron en verde.
- El build regenero `admin.html`, `admin.js` y el chunk canonico de admin.

## Gate remoto

- `npm run gate:admin:rollout:openclaw` no pudo ejecutarse en este entorno porque `powershell` no esta instalado.
- Se ejecuto la variante equivalente `npm run gate:admin:rollout:openclaw:node`.
- El gate remoto confirmo `admin.html`, CSP y contrato OpenClaw publicados.
- El gate remoto fallo en los smokes de `admin-v3-runtime` y `admin-openclaw-auth` sobre `https://pielarmonia.com` porque el dominio publico todavia sirve la version anterior del login y aun no refleja este cambio local.

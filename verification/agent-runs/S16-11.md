# S16-11 — Guard de drift para OpenAPI OpenClaw

## Resumen

- Se agrego `bin/check-openapi-drift.js` para comparar la superficie runtime de OpenClaw registrada en `lib/routes.php` contra `openapi-openclaw.yaml` y las firmas publicas de `OpenclawController.php`.
- El guard usa una tabla de mapeo explicita `resource -> path/method` para fallar si aparece una ruta runtime nueva, si falta una operacion en el YAML o si la ruta apunta a un metodo distinto del esperado.
- Se conecto el guard a `CI` para que se ejecute cuando cambien la spec, el controlador, las rutas o el propio script.

## Archivos

- `bin/check-openapi-drift.js`
- `tests-node/check-openapi-drift.test.js`
- `tests-node/ci-openapi-drift-workflow-contract.test.js`
- `.github/workflows/ci.yml`

## Verificacion

```bash
node --check bin/check-openapi-drift.js
node --check tests-node/check-openapi-drift.test.js
node --check tests-node/ci-openapi-drift-workflow-contract.test.js
node bin/check-openapi-drift.js
node --test tests-node/check-openapi-drift.test.js tests-node/ci-openapi-drift-workflow-contract.test.js
git diff --check
```

## Resultado

- `node bin/check-openapi-drift.js` -> `✅ OpenClaw OpenAPI drift guard — 13 runtime routes and 13 OpenAPI operations aligned.`
- La fixture de prueba con una ruta runtime extra no mapeada devuelve `exit 1`, cubriendo el drift que pide la tarea.

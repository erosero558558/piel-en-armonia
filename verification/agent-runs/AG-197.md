# AG-197 - Turnero PHP contract runner repair

Fecha: 2026-03-11
Owner: Ernesto
Executor: codex

## Objetivo

Reparar la ejecucion local del contrato PHP del turnero para que el frente vuelva a tener una ruta oficial de validacion sobre `AdminDataAppDownloadsContractTest.php`, incluso cuando `vendor/bin/phpunit` arranca con autoload de Composer desalineado en este workspace.

## Cambios consolidados

- Se agrego `bin/run-phpunit.js` como wrapper canonico para PHPUnit.
- El wrapper detecta drift entre `vendor/autoload.php` y `vendor/composer/autoload_real.php`, intenta reparar con `composer dump-autoload --no-scripts --no-interaction` y reintenta la corrida.
- El wrapper inyecta `--no-coverage` por defecto para evitar el warning local de cobertura que convertia la corrida en exit code `1`.
- `package.json` ahora expone `npm run test:turnero:php-contract`, incorpora ese paso dentro de `npm run gate:turnero` y agrega la suite `tests-node/run-phpunit-wrapper.test.js` a `agent:test`.
- El runbook del release turnero documenta la ruta PHP oficial y la semantica del wrapper.

## Validacion ejecutada

- `node --test tests-node/run-phpunit-wrapper.test.js`
- `npm run test:turnero:php-contract`
- `npm run gate:turnero`

## Resultado

- La suite unitaria del wrapper paso `5/5`.
- `npm run test:turnero:php-contract` ejecuta `tests/Integration/AdminDataAppDownloadsContractTest.php` en verde: `1 test, 9 assertions`.
- `npm run gate:turnero` volvio a quedar verde con contratos Node, contrato PHP del turnero y las `53` pruebas Playwright del frente.
- La deuda documentada en `AG-190` sobre `vendor/bin/phpunit` roto queda cerrada para el flujo operativo del turnero a traves del wrapper canonico.

# AG-214 - Improve per-asset cache-header diagnostics in prod verify

## Summary

- Updated `bin/powershell/Common.Http.ps1` so `Test-AssetCacheHeaders` reports failures per asset instead of collapsing the whole batch into a generic `cache-header:assets` error.
- Added request-level diagnostics with asset name, URL, and normalized `request_error:*` detail.
- Extended the Node contract coverage in `tests-node/prod-ops-public-sync-contract.test.js`.

- The prod verify gate now identifies the exact asset causing the cache-header failure.

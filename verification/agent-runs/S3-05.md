# S3-05

## Resumen

Se implemento la puerta publica de preconsulta en `es/pre-consulta/` para
capturar leads clinicos desde la web y crear un caso de Flow OS en stage
`lead_captured`, con notificacion a frontdesk y soporte para fotos clinicas.

## Cambios

- Se agrego la pagina publica `es/pre-consulta/` con formulario monolingue en
  espanol y feedback de carga/envio.
- Se incorporo el flujo cliente para subir hasta 3 imagenes y enviar el intake
  a `patient-cases`.
- Se expuso `POST /api.php?resource=patient-cases` para intake publico, con
  validaciones, rate limit, creacion/reutilizacion de caso abierto y callback.
- Se ajusto la hidratacion del read model para preservar casos persistidos y
  detectar correctamente stages como `lead_captured`.
- Se enlazaron fotos reclamadas al caso en storage clinico privado.
- Se actualizo SEO/routing para incluir `/es/pre-consulta/` en sitemap,
  checklist y cobertura Playwright.

## Validacion ejecutada

```bash
php -l controllers/PatientCaseController.php
php -l tests/Integration/PublicPreConsultationIntakeTest.php
npm run content:public-v6:validate
npm run build:public:v6
npx playwright test tests/pre-consulta-public-intake.spec.js --workers=1
npx playwright test tests/public-seo-routing.spec.js -g "route /es/pre-consulta/" --workers=1
php /tmp/preconsulta-harness.php
```

Resultados:

- Sintaxis PHP verde en controller e integracion.
- `content:public-v6:validate` verde.
- `build:public:v6` verde y genera `/es/pre-consulta/index.html`.
- Playwright del intake publico verde.
- Playwright focalizado del routing SEO para `/es/pre-consulta/` verde.
- Harness PHP directo verde, verificando `201`, `lead_captured`, callback y
  reclamo de fotos a storage clinico privado.

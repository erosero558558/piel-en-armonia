# S7-31 — Env & secrets inventory

Fecha: `2026-03-30`

## Objetivo

Auditar las variables de entorno usadas por el runtime real (`lib/` + `controllers/`), compararlas con [env.example.php](/Users/luciaguadalupecaizasanchez/Documents/GitHub/Aurora-Derm/env.example.php) y dejar inventario explícito de documentadas, compat legacy y defaults peligrosos.

## Comandos usados

```bash
rg -o -N "(?:app_env|getenv)\(\s*['\"]([A-Z0-9_]+)['\"]" lib controllers
node - <<'NODE'
const fs=require('fs');
const {execSync}=require('child_process');
const codeOut=execSync("rg -o -N \"(?:app_env|getenv)\\(\\s*['\\\"]([A-Z0-9_]+)['\\\"]\" lib controllers",{encoding:'utf8'});
const code=[...new Set([...codeOut.matchAll(/['\\\"]([A-Z0-9_]+)['\\\"]/g)].map(m=>m[1]))].sort();
const src=fs.readFileSync('env.example.php','utf8');
const doc=[...new Set([...src.matchAll(/putenv\\(\\s*['\\\"]([A-Z0-9_]+)=/g)].map(m=>m[1]))].sort();
console.log({ code: code.length, documented: doc.length });
NODE
php -l env.example.php
node bin/gate.js S7-31
```

## Resultado

- `248` nombres únicos detectados en código.
- `142` variables canónicas de producto/runtime quedan documentadas en `env.example.php`.
- `99` aliases legacy y `7` variables de sistema/runtime (`HOME`, `USERPROFILE`, `APP_ENV`, etc.) quedan inventariadas en [docs/ENV_INVENTORY.md](/Users/luciaguadalupecaizasanchez/Documents/GitHub/Aurora-Derm/docs/ENV_INVENTORY.md) sin promoverlas como happy path.
- Se añadieron al example los grupos faltantes de:
  admin agent relay, pricing/catalog, Redis/Clarity/Papertrail, OpenAI directo, compat backup/operator auth y compat avanzada Figo/OpenClaw.

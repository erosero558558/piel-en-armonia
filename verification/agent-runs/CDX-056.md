# CDX-056

## Resumen

- Slice frontend canónica del frente
  `STRAT-2026-03-turnero-web-pilot-local-first`.
- Reemplaza la slice local `CDX-053` y reaterriza en el canónico el replay de
  surfaces web y runtime del piloto `web_pilot`.
- El objetivo es que admin, operador, kiosco y sala lean el perfil clínico
  `web_pilot` como salida local válida, sin desktop/Android como blockers.

- `content/turnero/clinic-profile.json`
- `content/turnero/clinic-profiles/piel-armonia-quito.json`

- `js/queue-operator.js`
- `tests/admin-queue.spec.js`

## Cambios aplicados

- El perfil activo y el catalogado de `Piel Armonia Quito` pasan a
  `release.mode=web_pilot`, con `admin_mode_default=basic`,
  `separate_deploy=true` y `native_apps_blocking=false`.
- El install hub del admin deja de tratar desktop/Android como blockers del
  corte local-first y los presenta como carriles posteriores.
- El executor compartido del release pack tolera storages parciales sin API
  completa de `Storage`, evitando falsos rojos cuando el entorno solo inyecta
  una superficie parcial para persistencia local.
- Se regeneran los artefactos publicados del runtime desde fuente
  (`admin.js`, `js/queue-operator.js`) en lugar de resolver por merge manual.
- La suite Playwright de `admin-queue` queda alineada al contrato
  `web_pilot`, incluyendo heartbeats locales, rutas canónicas y el hub
  operativo del piloto.

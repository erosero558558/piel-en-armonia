# AG-162 — Harden shell V6 widget render contracts

## Resumen
- Se eliminaron fallbacks silenciosos en widgets del shell V6 que convertian contenido faltante en `''` o `'#'`.
- Se filtraron cards, links y bloques incompletos antes de renderizar.
- Se mantuvo intacto el layout, el runtime y los hooks `data-v6-*`.

## Archivos
- `src/apps/astro/src/components/public-v6/BookingStatusV6.astro`
- `src/apps/astro/src/components/public-v6/CorporateMatrixV6.astro`
- `src/apps/astro/src/components/public-v6/EditorialGridV6.astro`
- `src/apps/astro/src/components/public-v6/NewsStripV6.astro`
- `src/apps/astro/src/components/public-v6/PublicFooterV6.astro`
- `src/apps/astro/src/layouts/PublicShellV6.astro`
- artefactos estaticos `es/**` y `en/**`
- `verification/public-v6-canonical/artifact-drift.json`

## Cambios clave
- `BookingStatusV6`: no renderiza CTA ni seccion vacia si faltan datos reales.
- `CorporateMatrixV6`: filtra cards incompletas y evita CTA/categoria vacios.
- `EditorialGridV6`: normaliza `size/type`, filtra cards invalidas y evita metadata muda.
- `NewsStripV6`: deja de fabricar enlaces `#`, oculta toggle/panel si no existe contenido util y no muestra switch vacio.
- `PublicFooterV6`: filtra columnas, policies y copy vacios antes de renderizar.
- `PublicShellV6`: skip link y back-to-top solo existen si su label esta presente.

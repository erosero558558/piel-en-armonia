# S13-01 — robots.txt hardening

## Resultado

`robots.txt` ya estaba endurecido en `main` con todos los `Disallow` pedidos por la tarea:

- `/lib/`
- `/templates/`
- `/backup/`
- `/bin/`
- `/store/`

La tarea seguía abierta por drift del board, no por ausencia del hardening en código.

## Evidencia

Archivo verificado:

- `robots.txt`

Checks corridos:

```bash
sed -n '1,220p' robots.txt
rg -n "Disallow: /lib/|Disallow: /templates/|Disallow: /backup/|Disallow: /bin/|Disallow: /store/" robots.txt
npx playwright test tests/public-seo-routing.spec.js --grep "robots.txt keeps sitemap pointer and protects non-public surfaces" --workers=1
node bin/gate.js S13-01
```

## Nota

Se amplió la spec `tests/public-seo-routing.spec.js` para fijar explícitamente los `Disallow` del audit y evitar una reapertura por drift documental.

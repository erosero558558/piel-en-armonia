# AG-163 — Harden V6 header render contracts

## Resumen
- Se endurecio `PublicHeaderV6` para depender solo de datos explicitos y dejar de fabricar estado util desde fallbacks silenciosos.
- Se eliminaron rutas `#`, labels vacios y construccion implicita del buscador a partir de `links` o `mega`.
- El bloque se mantuvo en lane frontend puro; no se toco `bin/` ni tests cross-domain por guardrails del board.

## Archivo
- `src/apps/astro/src/components/public-v6/PublicHeaderV6.astro`

## Cambios clave
- `header.links` se sanea y filtra antes de renderizar.
- `mega.columns/items/children` se normalizan y solo renderizan si cumplen contrato minimo.
- El buscador usa solo `header.searchEntries` explicito; ya no se autoconstruye desde navegacion.
- `contact`, `switch`, `search`, `drawer`, `mega` y `brand` solo renderizan si sus datos requeridos existen.
- El drawer solo muestra footer/mega cuando esos bloques tienen contenido real.
- El dialogo de search solo existe si hay index, labels y aria suficientes para operar.

## Validacion
- `npm run build:public:v6`
- `npm run test:frontend:qa:v6`
- `npm run audit:public:v6:sony-parity`

## Resultado
- Todo en verde.
- La paridad Sony se mantuvo en `50/50`.
- Los artefactos `es/**` y `en/**` quedaron regenerados y sincronizados.

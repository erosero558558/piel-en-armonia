## AG-223 - Consolidar auth mock legacy en admin-queue spec

### Resumen

- `tests/admin-queue.spec.js` dejó de usar el wrapper local `installQueueAdminAuthMock` y ahora consume `installLegacyAdminAuthMock` directamente desde `tests/helpers/admin-auth-mocks.js`.
- `tests-node/admin-auth-mocks-helper.test.js` quedó extendido para fijar que `admin-queue`, `queue-operator` y `operator-auth-shared-session` reutilicen el helper compartido sin redefinir mocks locales.

### Verificacion

- `npx eslint tests/admin-queue.spec.js tests-node/admin-auth-mocks-helper.test.js`
    - Resultado: sin errores; quedaron solo warnings previos en `tests/admin-queue.spec.js`.
- `node tests-node/admin-auth-mocks-helper.test.js`
    - Resultado: `7/7` en verde.
- `npx playwright test tests/admin-queue.spec.js --grep "cola muestra guia accionable para apoyos activos y enfoca el ticket correcto|numpad cross-platform soporta Enter por location=3 y decimal regional|admin expone intento y causa cuando una desktop operador queda reintentando apertura|admin mantiene visible launchMode y alerta autoarranque apagado en desktop operador|admin muestra progreso de auto-update del operador sin perder el contexto operativo" --workers=1`
    - Resultado: `5/5` en verde.

### Resultado

- El spec de `admin-queue` ya no mantiene un alias legacy innecesario para auth.
- Queda fijado por contrato que el helper compartido de auth siga siendo la única fuente para estos specs.

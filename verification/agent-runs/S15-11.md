# S15-11 — Dispatch WIP limit

## Resumen

- `bin/dispatch.js` ahora calcula el WIP activo por rol/lane antes de recomendar una nueva tarea.
- Cada rol tiene un `wipLimit` configurable, con override por `--wip-limit` o variables de entorno `DISPATCH_WIP_LIMIT_*`.
- Cuando el lane ya está saturado, `dispatch` deja de sugerir otra tarea y muestra qué claims activas están ocupando ese cupo.

## Archivos

- `bin/dispatch.js`
- `tests-node/dispatch-cli.test.js`

## Validación

- `node --check bin/dispatch.js`
- `node --test tests-node/dispatch-cli.test.js`
- `node bin/dispatch.js --role backend`
- `git diff --check`

## Resultado

- El contrato cubre el caso de backend saturado por WIP y el caso de override explícito del límite.
- El CLI real sigue funcionando en el repo actual y solo bloquea si el lane correspondiente alcanza el máximo de claims activas.

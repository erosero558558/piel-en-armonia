# AG-218 - Teach repair-git-sync to classify GitHub deploy blockers and prefer self-hosted fallback

## Summary

- Updated `.github/workflows/repair-git-sync.yml` so the transport-fallback classifier now understands verify assets `github-deploy-*` in addition to stale-host signals.
- GitHub-hosted transport is no longer recommended when verify reports `github-deploy-transport-blocked`, `github-deploy-connectivity-blocked`, or `github-deploy-repair-git-sync-blocked`.
- The same classifier now emits `self_hosted_fallback_recommended` and `transport_fallback_github_blockers`, allowing repair to prefer the self-hosted lane when GitHub-hosted transport is blocked but the runner path is still viable.
- Updated the workflow contract in `tests-node/deploy-hosting-workflow-contract.test.js` and documented the new repair escalation in `docs/PUBLIC_MAIN_UPDATE_RUNBOOK.md`.

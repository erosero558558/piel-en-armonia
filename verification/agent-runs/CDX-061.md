## Resumen

- Slice `frontend` primaria del frente
  `STRAT-2026-03-turnero-web-pilot-multi-clinic-local`.
- Expone a `clinica-norte-demo` como segunda clínica `web_pilot` dentro del
  admin local-first, sin cambiar el perfil activo versionado

## Cambios aplicados

- `clinica-norte-demo` pasa a `release.mode=web_pilot` con
  `admin_mode_default=basic`, `separate_deploy=true` y
  `native_apps_blocking=false`.
- El store/admin runtime ahora consume `turneroClinicProfiles` y
  `turneroRegionalClinics` para mostrar catálogo y readiness multi-clínica.
- El install hub y el modelo del piloto local-first muestran a
  `clinica-norte-demo` como segunda clínica local-ready, manteniendo la
  publicación remota diferida.
- La spec `admin-queue` deja de exigir una salida remota lista y valida la
  capa avanzada local-first coherente con el contrato multi-clínica actual.
- El runtime publicado se regeneró desde fuentes para reflejar el cambio en
  `admin.js` y `js/queue-operator.js`.

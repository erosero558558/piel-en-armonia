# S7-08 — backup y restore automatizado

## Resumen

- Se agrego `ops/backup.sh` como wrapper operativo para generar snapshots horarios cifrados y comprimidos en `data/backups/YYYY-MM-DD-HH.json.gz`.
- El wrapper rota automaticamente snapshots horarios con mas de 7 dias.
- Se expuso el comando `npm run backup`.
- `bin/restore-backup.php` ahora acepta backups `.json.gz` ademas de `.json`.
- Se documento el flujo canonico en `docs/RUNBOOKS.md` (el repo no tiene `docs/RUNBOOK.md`; `RUNBOOKS.md` es la superficie canónica actual).

## Archivos

- `ops/backup.sh`
- `package.json`
- `bin/restore-backup.php`
- `tests-node/backup-script.test.js`
- `docs/RUNBOOKS.md`

## Verificacion

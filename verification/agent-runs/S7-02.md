# S7-02 — Hardening k8s secret example

## Resumen

- Se endurecio `k8s/secret.yaml.example` para reemplazar placeholders ambiguos
  por sentinels explicitos `__REPLACE_*__`.
- Se agrego `ops/check-secrets.sh` para leer el Secret real con `kubectl get secret`
  y fallar si detecta placeholders obvios, valores vacios o sentinels del template.
- Se incluyo el preflight en `docs/RUNBOOKS.md` y `docs/DEPLOY_HOSTING_PLAYBOOK.md`.
- Se dejo contrato automatizado en `tests-node/check-secrets-script.test.js`.

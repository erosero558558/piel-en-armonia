## Implementacion realizada

- Se creo la slice transversal `CDX-010` para reclamar:
    - `AGENT_BOARD.yaml`
    - `AGENT_HANDOFFS.yaml`
    - `PLAN_MAESTRO_CODEX_2026.md`
    - `verification/agent-runs/CDX-009.md`
    - `verification/agent-runs/CDX-010.md`
- Se alineo el mirror `board/plan` para que el workspace dejara de clasificar
  la evidencia de `CDX-009` como `unknown_scope`.
- Se conservo `CDX-009` como frente backend primario del incidente.
- `HO-001` queda como handoff cerrado historico.
- `HO-002` se uso como handoff transitorio para sostener la cobertura
  cross-domain mientras `CDX-010` permanecio en `review`, y luego quedo
  cerrado antes del cierre final.
- El `close` canonico sobre una tarea `cross_domain=true` quedo atrapado en una
  doble precondicion del propio orquestador:
    - el preflight de `publish` exige que la tarea siga activa y conserve un
      handoff `active` para que `codex-check` pase;
    - el cierre terminal exige que ya exista al menos un handoff `closed`
      vinculado.
- La salida operativa soportada por la CLI fue `codex stop CDX-010 --to done`,
  que deja el board y el plan en estado terminal correcto con los handoffs ya
  cerrados, listo para publicar el diff resultante.

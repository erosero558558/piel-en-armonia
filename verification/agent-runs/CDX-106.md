# CDX-106

## Resumen

- Slice `codex_frontend` del frente
  `STRAT-2026-03-turnero-web-pilot-multi-clinic-local` para cerrar
  `multi_clinic_local_readiness`.
- Corrige el renderer del piloto `admin-queue` para que no dependa de un DOM

## Cambios aplicados

- `renderQueueOpsPilotView()` ahora consulta grupos `details` solo si el host
  expone `querySelectorAll()`, y degrada a `[]` en stubs minimos de Node.
- La deteccion de grupos abiertos deja de depender exclusivamente de
  `HTMLDetailsElement`; usa un guard compatible con entornos sin ese
  constructor global.
- Se incorpora `#queueOpsPilotCompactControlTowerHost` en la columna visible
  del piloto, entre `Rutas por clinica` y `Validacion operativa`.
- `turnero-release-control-tower.js` soporta una variante `compact` que reuse
  el mismo modelo multi-clinica y las mismas acciones de copia/JSON, mientras
  conserva intacta la tarjeta completa del host avanzado.
- La consola pesada sigue dentro de `queueOpsPilotAdvancedGroup` y mantiene el
  orden canonico:
    - `queueOpsPilotRolloutGovernorHost`
    - `queueOpsPilotExecutivePortfolioStudioHost`
    - `queueOpsPilotStrategyDigitalTwinStudioHost`
    - `queueMultiClinicControlTowerHost`
    - `queueReleaseBoardOpsHubHost`
    - `queueReleaseOpsConsoleHost`
    - `queueSurfaceAcceptanceConsoleHost`
    - `queueReleaseMissionControlHost`

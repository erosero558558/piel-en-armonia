# S3-43

## Resumen

Se agregó la emisión de certificados médicos dentro del workspace de historia
clínica en admin. El médico ahora puede abrir un modal desde el caso activo,
completar el certificado, emitirlo vía API y ver de inmediato el folio, el
PDF y el acceso rápido a WhatsApp.

## Cambios

- Se añadió el botón `📋 Certificado` y un host de overlay en la vista de
  historia clínica de admin.
- Se implementó un modal de emisión con tipo de certificado, días de reposo,
  diagnóstico con opciones CIE-10, restricciones y observaciones.
- El flujo hace `POST /api.php?resource=certificate`, muestra el folio
  devuelto, el enlace PDF y el CTA de WhatsApp dentro del mismo modal.
- Tras emitir, el estado del caso se sincroniza con la HCE mediante
  `clinical-episode-action` usando `issue_certificate`, con fallback local si
  esa sincronización falla.
- Se corrigió además el boot del workspace clínico importando
  `renderClinicalCompareFlow`, que ya era usado por la sección y estaba
  rompiendo la carga del admin publicado.
- Se actualizó la cobertura Playwright del workspace clínico para validar la
  emisión del certificado end-to-end con mocks.

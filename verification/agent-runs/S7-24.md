# S7-24 — Auditoría `app-downloads/` y `desktop-updates/`

Fecha: 2026-03-30

## Resumen

- Se creó `docs/DESKTOP_DISTRIBUTION.md` con inventario, consumidores y
  riesgos del canal desktop.
- `app-downloads/index.php` es público; no exige auth propia y Caddy lo expone
  por rewrite directo.
- El único canal realmente publicado en este repo hoy es `pilot/operator/win`.
- `kiosk` y `sala_tv` salen desde defaults del registry, no desde manifiestos
  publicados.
- Los checksums del bundle `operator/win` son consistentes entre manifest,
  `SHA256SUMS.txt`, `app-downloads/` y `desktop-updates/`.

## Validación ejecutada

```bash
find app-downloads -maxdepth 4 -type f | sort
find desktop-updates -maxdepth 5 -type f | sort
php -l app-downloads/index.php
php -r 'require "lib/AppDownloadsCatalog.php"; echo json_encode(read_app_downloads_catalog(), JSON_PRETTY_PRINT|JSON_UNESCAPED_SLASHES), PHP_EOL;'
grep -rn 'API_KEY\|password\|secret\|sk_live' release/ 2>/dev/null
shasum -a 256 \
  release/turnero-apps-pilot-local/app-downloads/pilot/operator/win/TurneroOperadorSetup.exe \
  release/turnero-apps-pilot-local/desktop-updates/pilot/operator/win/TurneroOperadorSetup.exe
npm run turnero:verify:pilot:local
```

## Hallazgos

- `app-downloads/index.php` sirve el centro público de instalaciones.
- `ops/caddy/Caddyfile` reescribe `/app-downloads` a `/app-downloads/index.php`
  y lo trata como `publicPhp`.
- `read_app_downloads_catalog()` resuelve:
  - `operator`: `version=0.1.0`, `updatedAt=2026-03-13T06:23:55.502Z`,
    manifest real `pilot`
  - `kiosk`: defaults `stable`, sin `updatedAt`
  - `sala_tv`: defaults `stable`, sin `updatedAt`
- El hash del instalador Windows coincide en `app-downloads/` y
  `desktop-updates/`:
  `42be7ebb5bf5442d5df8f51c4dbbd4ffac6ee584a45c36038115bf5ed95ed5cf`

## Riesgo residual

- `npm run turnero:verify:pilot:local` falla con `MODULE_NOT_FOUND` porque
  falta `bin/verify-turnero-release-bundle.js`.
- La UI de `app-downloads/` tiene probes de readiness, pero el payload sigue
  mezclando surfaces publicadas con defaults del registry. Eso permite que
  `kiosk` y `sala_tv` aparezcan en el catálogo aunque no tengan artefacto real
  en este repo.

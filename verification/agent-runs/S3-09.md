# S3-09 - Vista expandida del operador

## Resumen

- Se agrego una tarjeta de foco clinico para el operador con paciente, motivo, visitas previas, etapa y alertas.
- Se enriquecio el read model del queue para exponer `patientLabel`, `visitReasonLabel`, `priorVisitsCount`, `journey*` y `operatorAlerts` en backend y frontend.
- Se reparo `lib/FlowOsJourney.php`, que en esta rama estaba roto de base: habia un bloque truncado con parse error, helpers faltantes y stages canonicos mezclados con el manifest runtime.
- Se ajusto `QueueService` para preservar alertas inline de asistencia aunque no exista un `help_request` activo separado.

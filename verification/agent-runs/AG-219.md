# AG-219 - Restore admin rollout gate policy compatibility

Fecha: 2026-03-13
Owner: codex_backend_ops
Dominio: deploy

## Objetivo

Restaurar el contrato operativo de `GATE-ADMIN-ROLLOUT.ps1` para que vuelva a
ser compatible con la politica central de admin rollout y con los workflows que
ya propagaban `stage`, `AllowFeatureApiFailure` y `AllowMissingAdminFlag`.

## Causa raiz

- El run `Post-Deploy Fast Lane` `23029725256` fallaba en `Ejecutar gate admin UI rollout (fast)`.
- Error exacto observado:
  `Cannot validate argument on parameter 'Stage'. The argument "canary" does not belong to the set "stable"`.
- El script canónico `scripts/ops/admin/GATE-ADMIN-ROLLOUT.ps1` habia quedado reducido a `Stage=stable` y habia perdido los switches `AllowFeatureApiFailure` y `AllowMissingAdminFlag`.
- `package.json` tambien habia quedado con `npm run gate:admin:rollout -Stage stable`, asi que el drift no era solo del workflow sino tambien del entrypoint operativo local.

## Cambio aplicado

- Se restauro en `scripts/ops/admin/GATE-ADMIN-ROLLOUT.ps1` el contrato stage-aware:
  `internal|canary|general|rollback`.
- Se reintrodujeron los switches `AllowFeatureApiFailure` y
  `AllowMissingAdminFlag`, con guardrail efectivo para `internal`.
- El gate vuelve a validar `admin_sony_ui` y `admin_sony_ui_v3` contra
  `/api.php?resource=features` segun la etapa solicitada.
- Se preservaron las mejoras mas nuevas del shell canonico:
  chequeo de `admin-v3.css`, `admin.js`, ausencia de CSS legacy, CSP endurecida,
  `TEST_BASE_URL` y `TEST_REUSE_EXISTING_SERVER=0` para Playwright.
- Se dejaron chequeos de URLs inertes (`admin_ui=legacy|sony_v2|sony_v3`,
  `admin_ui_reset=1`) para asegurar compatibilidad pasiva del front door.
- Se alineo `package.json` con atajos canónicos:
  `gate:admin:rollout`, `:internal`, `:canary`, `:general`, `:rollback`.
- Se actualizo `docs/RUNBOOKS.md` con las etapas y el comportamiento efectivo
  del gate.
- Se actualizo `tests-node/admin-rollout-gate-contract.test.js` para fijar el
  nuevo contrato.

# AG-219 - Cancel stale self-hosted fallback queue on repair recovery

## Summary

- Updated `.github/workflows/repair-git-sync.yml` so the repair lane now reconciles queued `deploy-frontend-selfhosted.yml` runs after every self-hosted fallback evaluation.
- When repair no longer recommends the self-hosted path, it now cancels stale queued fallback runs from older attempts so a late Windows runner cannot publish an obsolete commit.
- When repair still needs the self-hosted path and a new fallback remains queued, it now preserves the current candidate and cancels older queued duplicates.
- Surfaced the new queue reconciliation telemetry in the self-hosted incident body and in the repair summary:

  - `self_hosted_queue_cancelled_count`
  - `self_hosted_queue_cancelled_ids`

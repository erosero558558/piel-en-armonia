# AG-253 - HealthVisibilityTest alineado con publicSync publico

## Resumen

Se alineo el contrato de `tests/Integration/HealthVisibilityTest.php` con el
payload publico actual de `HealthController`. El health publico ya no se trata
como un payload minimo sin `checks`: ahora expone un resumen seguro de
`checks.publicSync` y conserva los campos publicos de calendario sin filtrar
rutas internas.

## Cambios validados

- `tests/Integration/HealthVisibilityTest.php` ahora exige:
    - presencia de `calendarConfigured`, `calendarReachable`, `calendarMode` y
      `calendarSource`
    - presencia de `checks.publicSync`
    - ausencia de surfaces internos como `storage`, `auth` e `internalConsole`
- `controllers/HealthController.php` sigue publicando solo un resumen seguro de
  `publicSync`, sin exponer `repoPath`, `statusPath`, `logPath`, `lockFile` ni
  arrays internos completos.

## Verificacion

Comandos ejecutados localmente el 14 de marzo de 2026:

```bash
php vendor/bin/phpunit --coverage-text --filter HealthVisibilityTest
php vendor/bin/phpunit --coverage-text --coverage-clover coverage.xml
npm run agent:test
```

Resultados observados:

- `HealthVisibilityTest` en verde
- suite PHPUnit en verde (`211/211`)

## Resultado operativo

El fallo de CI ligado al contrato del health publico queda corregido en repo.
Lo que siga pendiente para `public_main_sync` ya no pertenece a este task de
tests, sino al rollout externo del host.

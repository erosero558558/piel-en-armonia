# S9-01 — Portal: próxima cita viva

## Resultado

- Se creó la surface real del portal en `es/portal/`, `es/portal/login/` y `es/portal/historial/`.
- El frontend dejó de usar `P-TEST` y ahora consume `GET /api.php?resource=patient-portal-dashboard`.
- La card de próxima cita muestra fecha, hora, doctor, tipo, preparación, CTA de WhatsApp y botón de reagendamiento.
- Cuando no existe una cita futura, el portal muestra un estado vacío con CTA destacado para agendar.

## Backend

- `controllers/PatientPortalController.php`
  - nuevo método `dashboard()`
  - resumen autenticado de próxima cita
  - links de soporte y reagendamiento
- `lib/PatientPortalAuth.php`
  - nuevo helper `authenticateSession()`
  - helper público para match de teléfono del paciente
- `lib/routes.php`
  - alta de `patient-portal-dashboard`
- `lib/ApiConfig.php`
  - rate limit y allowlist pública para el endpoint

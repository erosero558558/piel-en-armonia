# AG-010 - Funnel drop-off instrumentation and admin visibility

## Goal
Expose actionable drop-off reasons (`reason`/`error_code`) per funnel step for weekly analysis, and make admin data consumption more consistent.

## Implemented changes
- Refactored `controllers/AnalyticsController.php`:
  - Added `AnalyticsController::buildFunnelMetricsData(array $context): array`.
  - `getFunnelMetrics()` now reuses the shared builder.
- Updated `controllers/AdminDataController.php`:
  - `/api.php?resource=data` now includes `data.funnelMetrics` (same contract shape as `/funnel-metrics`).
  - Added resilient fallback (`funnelMetrics: null`) if metrics export fails.
- Updated `src/apps/admin/modules/data.js`:
  - Admin first consumes `data.funnelMetrics` from `/data`.
  - Keeps fallback request to `/funnel-metrics` if embedded metrics are missing.

## Validation
- PHP syntax:
  - `php -l controllers/AnalyticsController.php` -> OK
  - `php -l controllers/AdminDataController.php` -> OK
- Build:
  - `npm run build` -> OK
- E2E focused tests:
  - `npx playwright test tests/funnel-tracking.spec.js tests/funnel-retention-admin.spec.js tests/admin-availability-readonly.spec.js`
  - Result: `6 passed, 1 skipped`

## Outcome
- Drop-off dimensions remain visible (`checkout_step`, `reason`, `error_code`) and now are available to admin through `/data` without requiring an extra mandatory fetch.

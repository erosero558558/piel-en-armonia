# S15-08 — Agent health check

## Resumen

- Se creó `bin/health-check.js` como doctor rápido del sistema de agentes con salida texto/JSON y semáforo por check.
- El script valida cinco superficies: regex canónica de task IDs, sync de `BACKLOG.md`, cobertura de `sitemap.xml`, conteo de `governance/broken-scripts.json` y convergencia entre `bin/report.js` y `agent-orchestrator.js status --json`.
- Se añadió el comando `npm run agent:health` y un contrato Node que cubre el escenario totalmente verde.

## Archivos

- `bin/health-check.js`
- `bin/dispatch.js`
- `bin/sync-backlog.js`
- `package.json`
- `tests-node/health-check-cli.test.js`

## Validacion

- `node --check bin/health-check.js`
- `node --check bin/dispatch.js`
- `node --check bin/sync-backlog.js`
- `node --test tests-node/health-check-cli.test.js`
- `node bin/health-check.js --json`
- `npm run agent:health`
- `git diff --check`

## Resultado

- El doctor deja pasar con exit `0` solo cuando todos los checks estan en verde.
- `bin/sync-backlog.js --check` ahora ignora el timestamp generado para no marcar falso drift.
- En este repo el rojo residual visible queda limitado a deuda heredada en `sitemap.xml` y `governance/broken-scripts.json`, no a drift del corte.

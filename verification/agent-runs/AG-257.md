# AG-257

## Alcance

- Habilitar `publish checkpoint` para tareas `AG-*` y mantener compatibilidad con `CDX-*`.
- Introducir la excepcion formal `validated_release_promotion` para promocionar releases ya validados desde scopes bloqueados por estrategia.
- Dejar diagnostico warn-first cuando un push a `main` sale bien pero `public_main_sync` todavia no confirma deploy.

## Cambios

- Se agrego `isReleasePromotionExceptionTask()` y el bypass estricto de `blocked_scopes` solo para tareas `review` con `strategy_role=exception`, `strategy_reason=validated_release_promotion`, `work_type=evidence`, `integration_slice=governance_evidence` y `executor=codex`.
- `task start --release-publish` ahora rellena el preset canonico de release publish y permite usarlo sobre tareas existentes `AG-*` o `CDX-*`.
- `publish checkpoint` ahora acepta `AG-*`, distingue `task_family`, devuelve `live_status`, `verification_pending` y `warning_code`, y trata la falta de confirmacion live como `pending` no bloqueante cuando el push a `main` ya salio.
- El gate de publish deja de ejecutar `focus check` por separado y usa la validacion de foco integrada en `publish checkpoint`, evitando bloqueos redundantes cuando el board historico todavia no tiene foco activo.
- `status --json` y `board doctor --json` ahora leen `verification/agent-publish-events.jsonl` y exponen warnings de publish live pendiente por lane.
- La documentacion canonica de gobernanza y publish se actualizo para reflejar el flujo nuevo.

## Validacion

- `node --test tests-node/orchestrator/task-guards.test.js tests-node/agent-orchestrator-cli.test.js tests-node/publish-checkpoint-command.test.js tests-node/orchestrator/diagnostics.test.js`
- `php bin/validate-agent-governance.php`
- `npm run agent:policy:lint`

## Resultado

- El contrato de release publish queda cubierto en Node y PHP.
- `publish checkpoint` ya no queda amarrado a `CDX-*`.
- Un release validado puede promoverse sin romper la estrategia activa normal.
- La falta de confirmacion inmediata de `public_main_sync` deja warning visible sin convertir un push exitoso en fallo total.

# S9-08 — Lead scoring operativo

## Resumen

- Se creó `lib/lead/LeadScoringService.php` como capa canónica para puntuar leads de 0 a 100.
- `LeadOpsService` ahora delega el score comercial en ese servicio y persiste `heuristicScore`, `priorityBand`, `scoreSummary` y `scoreFactors`.
- La UI de callbacks en admin muestra el score junto al nombre del lead y una síntesis corta de factores.
- El runtime publicado quedó recompilado en `admin.js` y `js/queue-operator.js`.

## Factores del score

- Urgencia clínica
- Valor estimado del servicio
- Historial de `no_show`
- Canal / source / surface
- Servicio premium
- Tiempo en cola como ajuste operativo

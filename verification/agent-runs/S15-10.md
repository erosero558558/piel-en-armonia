# S15-10 — Report regressions section

## Resumen

- `bin/report.js` ahora incorpora la sección `Regresiones sospechosas`.
- La sección usa `regression-watch.js` para listar hallazgos cuando existen, o mostrar un estado explícito de repo sano cuando no hay regresiones sospechosas.
- El reporte sigue saliendo con exit `0` aunque haya hallazgos del watchdog; la idea es informar, no romper el resumen diario.

- `tests-node/report-regressions-cli.test.js`

- `git diff --check`

## Resultado

- `npm run report --silent` ya muestra la nueva sección sin errores.
- La prueba CLI cubre un repo temporal con un archivo borrado referido por una tarea `[x]` y otro repo temporal sin regresiones sospechosas.

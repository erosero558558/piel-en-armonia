# AG-250 - Public sync rollout bloqueado por host stale

## Resumen

Al 14 de marzo de 2026 el trabajo de repo ya esta listo, pero el host
publico sigue sin reflejar el `HealthController` actualizado. Por eso
`public_main_sync` continua en `registry_only/unverified` aunque el contrato
local ya este cubierto.

## Evidencia viva

### 1. Verificacion del job

```bash
node agent-orchestrator.js jobs verify public_main_sync --json
```

Resultado observado:

- `verification_source=registry_only`
- `verified=false`
- `failure_reason=unverified`

### 2. Health publico

```bash
curl -sS https://pielarmonia.com/api.php?resource=health
```

Payload observado el 14 de marzo de 2026:

```json
{"ok":true,"status":"ok","storageReady":true,"dataDirWritable":true,"timingMs":13,"version":"20260314072914","timestamp":"2026-03-14T07:49:46-05:00"}
```

El payload sigue sin `checks.publicSync`, asi que el orquestador no puede
validar telemetria viva del job desde el host.

### 3. Capacidad operativa de este entorno

Comprobaciones locales:

- `env | rg "^(SSH_|FTP_|PROD_|STAGING_|GH_|GITHUB_)"` solo expone `GH_PAGER`
- binaries presentes: `ssh`, `sftp`
- `~/.ssh/config` solo contiene la entrada `github.com-piel`
- no hay alias de VPS ni credenciales SSH/FTP/GitHub Actions visibles para
  ejecutar un rollout seguro desde esta sesion
- `origin` local apunta a `git@github.com-piel:erosero558558/Aurora-Derm.git`
  mientras que `repair-git-sync.yml` y el runbook operativo esperan
  `erosero558558/piel-en-armonia`, asi que usar `git push` desde esta copia no
  es una forma segura de disparar el flujo correcto

## Conclusión operativa

`AG-250` ya no esta bloqueado por codigo del repo. Queda bloqueado por una de
estas dos acciones externas:

1. desplegar al host la version actual de `controllers/HealthController.php`
2. disparar desde un entorno con credenciales validas el flujo de repair/deploy
   que sincroniza `/var/www/figo` y publica el `health` actualizado

## Salida del bloqueo

El task puede volver a `in_progress` o cerrarse cuando se cumplan ambos puntos:

- `curl https://pielarmonia.com/api.php?resource=health` expone
  `checks.publicSync`
- `node agent-orchestrator.js jobs verify public_main_sync --json` deja de
  responder `registry_only/unverified`

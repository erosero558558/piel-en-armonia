# S15-01 — Velocity regex and June critical sprints

## Summary

- Refactored `bin/velocity.js` into testable helpers so the task parser and git-log parser share the same `S*` / `UI*` task-ID contract.
- Expanded the June critical slice from Sprint 3 only to `S3 + S8 + S9`, while keeping legacy Sprint 3 metrics in the payload for compatibility.
- Added parser coverage metrics to the JSON/text output: `uiTaskTotal` and `sprint8To14Total`.
- Added regression coverage in `tests-node/velocity-cli.test.js`.

## Validation

- `node --test tests-node/velocity-cli.test.js`
- `node --test tests-node/audit-cli.test.js`
- `node bin/velocity.js --json`
- `node bin/status.js --json`
- `git diff --check`

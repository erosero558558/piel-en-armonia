# AG-241 - Consolidar fallback mocks de queue-state en admin queue spec

## Resumen

- Se ajustó `installAdminQueueApiMocks(...)` para que el `handleRoute`
  personalizado pueda interceptar requests antes de que el router base consuma
  `/data`, `/queue-state` u otros recursos comunes.
- El cambio consolida el fallback de `queue-state` dentro del helper principal
  del spec y evita tener que duplicar rutas inline para escenarios puntuales.
- El contrato funcional del spec no cambió: el objetivo fue dejar el harness
  más flexible sin romper los flujos de cola, piloto web ni apps operativas.

- `tests/admin-queue.spec.js`
- `verification/agent-runs/AG-241.md`

## Resultado

El harness de `admin-queue` ahora permite sobreescribir fallbacks de
`queue-state` desde cada escenario sin pelearse con el router base del helper,
manteniendo intacta la cobertura integral del spec.

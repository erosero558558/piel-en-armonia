# S7-09 — k8s readiness/liveness probes

## Resumen

- Se agregaron `readinessProbe` y `livenessProbe` al contenedor `app` en `k8s/deployment.yaml`.
- Ambos probes apuntan al contrato canonico `GET /api.php?resource=health`.
- `readinessProbe` queda con `failureThreshold=3`.
- `livenessProbe` queda con `failureThreshold=5`.
- Se agrego contrato automatizado para congelar el YAML y un smoke local de latencia para el endpoint `health`.

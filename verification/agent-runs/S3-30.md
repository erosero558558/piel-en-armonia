# S3-30

## Resumen

Se implemento la nueva sala publica `es/telemedicina/consulta/` como una
superficie V6 para teleconsulta: sala de espera virtual, carga de video Jitsi
en la misma pagina, handoff a WhatsApp clinico y panel para compartir hasta 3
fotos seguras antes o durante la llamada.

## Cambios

- Nueva ruta publica `es/telemedicina/consulta/` en Astro.
- Nuevo componente V6 `TeleconsultationRoomPageV6.astro` con:
  sala de espera, contexto de sala por query params, embed de video bajo
  demanda, subida de fotos y CTA de WhatsApp enriquecido.
- Nuevo stylesheet `teleconsultation-room.css` para la experiencia premium de
  teleconsulta.
- La landing `/es/telemedicina/` ahora enlaza a la sala desde acciones utiles.
- Se agrego discoverability minima en `navigation.json`, `sitemap.xml`,
  checklist de produccion, SEO routing y taxonomy V6.
- Se sumo Playwright focalizado para la ruta y su flujo de fotos/WhatsApp.

## Validacion ejecutada

```bash
npm run content:public-v6:validate
npx playwright test tests/public-v3-entry-surface-taxonomy.spec.js -g "maps each public route" --workers=1
npx playwright test tests/teleconsulta-room.spec.js tests/public-seo-routing.spec.js -g "teleconsultation room|route /es/telemedicina/consulta/" --workers=1
npm run build:public:v6
npx playwright test tests/teleconsulta-room.spec.js --workers=1
```

Resultados:

- `content:public-v6:validate`: verde.
- Taxonomy V6: verde para la ruta nueva junto al resto de superficies canonicas.
- SEO routing focalizado: verde para `/es/telemedicina/consulta/`.
- `tests/teleconsulta-room.spec.js`: verde, cubriendo personalizacion por query
  params y handoff a WhatsApp tras subir fotos.
- `build:public:v6`: verde, genera `/es/telemedicina/consulta/index.html` y
  deja artifacts en sync.

## Notas

- La sala usa `transfer-proof` como superficie existente para subir fotos
  seguras y luego las pliega al mensaje de WhatsApp; no abre todavia el flujo
  de triaje/adjunte clinico profundo, que encaja mejor con `S3-31`.

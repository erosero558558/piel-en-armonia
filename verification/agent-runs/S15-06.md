# S15-06 — Detector de commits done sin claim

## Resumen

- `bin/sync-backlog.js` ahora revisa el diff actual de `AGENTS.md` para detectar tareas que pasaron de `[ ]` a `[x]`.
- Si una tarea marcada como done no tiene archivo de claim vigente ni evidencia de claim en el diff actual, el script emite un warning en stdout.
- Se añadió soporte `AURORA_DERM_ROOT` para validar el flujo sobre repos temporales y el alias `npm run sync:backlog`.

## Archivos

- `bin/sync-backlog.js`
- `package.json`
- `tests-node/sync-backlog-cli.test.js`

## Validacion

- `node --check bin/sync-backlog.js`
- `node --test tests-node/sync-backlog-cli.test.js`
- `AURORA_DERM_ROOT=<tmp> npm run sync:backlog --silent`
- `git diff --check`

## Resultado

- `sync-backlog` ahora advierte cuando alguien marca una tarea como done sin haber dejado evidencia de claim en el cambio.
- Si el mismo cambio incluye un release legitimo del claim, no se emite warning falso.

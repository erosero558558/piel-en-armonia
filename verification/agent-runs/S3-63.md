# S3-63

## Resumen

Se actualizó `CLAUDE.md` con una tabla de comandos oficiales day-1, centrada en
los comandos que un agente nuevo necesita para operar sin caer en rutas legacy.

## Cambios

- Se renombró la sección a `Tabla de comandos oficiales`.
- Se dejó la tabla en 10 filas, debajo del máximo pedido de 12.
- Se incluyeron explícitamente `status`, `dispatch`, `claim`, `release`,
  `gate`, `stuck`, `report`, `velocity`, `sync-backlog` y `conflict`.
- Cada fila ahora documenta el equivalente o práctica “NO usar”.

## Validacion ejecutada

```bash
rg -n "## Tabla de comandos oficiales|npm run status|node bin/velocity.js|node bin/conflict.js" CLAUDE.md
node - <<'NODE'
const fs = require('fs');
const md = fs.readFileSync('CLAUDE.md', 'utf8');
const section = md.split('## Tabla de comandos oficiales')[1].split('## Mapa del repo')[0];
const rows = section.split('\n').filter((line) => line.startsWith('| `'));
console.log(rows.length);
NODE
```

Resultados:

- `rg` confirma que la tabla quedó en `CLAUDE.md` con los comandos pedidos.
- El conteo de filas devuelve `10`, dentro del máximo de `12`.

## Notas

- Se eligió `CLAUDE.md` en lugar de `README.md` porque ya es el punto de entrada
  principal para agentes y evitamos duplicar la misma tabla en dos lugares.

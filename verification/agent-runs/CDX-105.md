## Canon operativo

- `AGENT_BOARD.yaml` manda sobre estrategia, foco, slices y estados.
- `PLAN_MAESTRO_CODEX_2026.md` queda como espejo narrativo derivado.
- La estrategia heredada `STRAT-2026-03-admin-operativo` deja de gobernar esta
  fase porque habia quedado activa sin slices vivas y con required checks de

- Se habilita una estrategia solo transversal de gobernanza; para permitirla,
  el validador deja de exigir required checks de producto o subfrentes
  frontend/backend cuando el frente es exclusivamente `codex_transversal`.

## Control-root canonico

- `workspace.js` deja de tratar el worktree de `root_branch=main` como root
  operativo obligatorio.
- Se introduce `control_root` como root operativo canonico y separado de:
  - `repo_root`: fallback/referencia del repo compartido
  - `current_root`: cwd real desde donde corre el comando
- Precedencia del allocator:
  1. worktree actual si contiene control plane (`agent-orchestrator.js`,
     `AGENT_BOARD.yaml`) y no es un task worktree gestionado
  2. otro worktree elegible del mismo repo
  3. fallback al root del branch canonico si no existe worktree elegible
- `.codex-local` y `.codex-worktrees` pasan a resolverse bajo `control_root`.
- El snapshot de workspace expone `repo_root`, `control_root`,
  `current_root`, `root_branch_root` y `control_root_selection`.
- `workspace_root_not_ready` se mantiene como codigo publico, pero ahora
  reporta el `control_root` efectivo y adjunta `workspace_snapshot` en JSON.

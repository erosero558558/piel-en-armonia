# S3-44

## Resumen

Se agrego el historial de certificados emitidos dentro de la seccion
"Documentos" del workspace clinico en admin. La vista ahora consulta
`GET /api.php?resource=certificate&case_id=X`, muestra los certificados
emitidos para el caso activo y ofrece descarga PDF por cada documento.

## Cambios

- Se extendio el estado de revision de historia clinica para hidratar y
  conservar `certificateHistory`, su estado de carga y los errores no
  bloqueantes del fetch.
- Se incorporo una carga dedicada del historial de certificados al abrir la
  session clinica, reutilizando el `case_id` del caso activo y manteniendo el
  resto de la review usable aunque la consulta falle.
- La pestaña "Documentos" ahora renderiza un bloque "Historial emitido" con
  tarjetas por certificado, incluyendo folio, tipo, fecha, metadatos clinicos
  y boton/enlace de descarga PDF.
- Se agregaron selectores estables para QA de admin y una prueba Playwright
  focalizada para verificar el historial y el enlace de descarga por
  certificado.
- El rebuild del runtime publicado actualizo `admin.js` y `js/queue-operator.js`
  para alinear los artefactos compilados con el bundle actual del workspace.

## Notas

- El historial usa el endpoint canonico ya existente de certificados, asi que
  no fue necesario ampliar backend para este corte.
- La descarga PDF se arma por `id` de certificado emitido y conserva el formato
  `resource=certificate&id=...&format=pdf` esperado por la API actual.

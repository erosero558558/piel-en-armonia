# S7-07 — Prometheus scraping real

## Resumen

- alineé el stack de monitoring local a Aurora Derm en `docker-compose.monitoring.yml` y `prometheus.docker.yml`
- añadí bearer token de diagnósticos para que Prometheus pueda scrapear `GET /api.php?resource=metrics` desde Docker sin depender de localhost real
- extendí `SystemController::metrics()` con gauges de cola (`queue_size`, waiting/called/help requests) derivados del mismo store que alimenta `queue-state`
- agregué alertas mínimas en `prometheus.rules.yml` para backlog de cola y `api_error_rate_5m > 5%`
- documenté el flujo canónico en `docs/MONITORING.md` y `docs/API.md`

- El task original menciona `queue-state`, pero Prometheus no puede scrapear JSON directamente. El corte lo resuelve exportando gauges de cola en `/api.php?resource=metrics` a partir del mismo `store` que usa `queue-state`, y deja esa relación documentada.

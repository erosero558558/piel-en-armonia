# AG-261 - OpenClaw auth contract mode preservation during CDX-009 triage

## Objetivo

Preservar `openclaw_chatgpt` como contrato publico de auth/readiness mientras
`CDX-009` sigue bloqueado por `502` host-side en `pielarmonia.com`.

## Cambio aplicado

Archivo tocado:

- `lib/auth.php`

Ajustes:

- `OPERATOR_AUTH_SOURCE` vuelve a apuntar a `openclaw_chatgpt`.
- `operator_auth_normalize_mode()` preserva `openclaw_chatgpt` como modo
  expuesto y acepta `openclaw` / `google` como aliases utiles.

## Validacion

- `php tests/test_operator_auth.php`
  - `6 passed, 0 failed`
- `php tests/test_admin_auth_openclaw_facade.php`
  - `9 passed, 0 failed`
- `node --test tests-node/openclaw-auth-rollout-diagnostic.test.js tests-node/prod-openclaw-auth-rollout-contract.test.js`
  - `11 passed, 0 failed`
- smoke puntual con `PIELARMONIA_OPERATOR_AUTH_MODE=openclaw_chatgpt`
  - `operator_auth_mode()=openclaw_chatgpt`
  - `operator_auth_recommended_mode()=openclaw_chatgpt`
  - `internal_console_primary_auth_mode()=openclaw_chatgpt`

## Relacion con CDX-009

Este fix corrige una regresion repo-side de contrato auth, pero no resuelve el
incidente principal: el dominio publico sigue devolviendo HTTP `502` incluso en
`/api.php?resource=health-diagnostics`, por lo que el bloqueo vivo de
`CDX-009` permanece del lado host/edge/origen.

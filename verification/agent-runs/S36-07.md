## S36-07

- El fallo real no estaba en la lógica de callbacks sino en el runtime publicado del admin.
- `admin.js` apuntaba a `js/admin-chunks/index-D_Wo4uZb.js`, pero ese chunk no existía en el repo publicado.
- Como resultado, el boot del admin abortaba antes de montar la sección `#callbacks`, por eso el grid nunca llegaba a hidratarse.
- Regeneré el runtime con `npm run build:turnero:runtime`, lo que publicó:
  - `admin.js`
  - `js/queue-operator.js`
  - `js/admin-chunks/index-BW4W8eSB.js`
- También abrí `.gitignore` para permitir trackear `js/admin-chunks/*.js`; sin eso, el siguiente publish podía volver a dejar `admin.js` apuntando a un chunk que Git no veía.

# CDX-057

## Resumen

- Slice backend/gates canónica del frente
  `STRAT-2026-03-turnero-web-pilot-local-first`.
- Reemplaza la slice local `CDX-054` y consolida contrato, readiness, scripts y
  runner local del carril `web_pilot`.
- El objetivo es validar el piloto web local por clínica sin arrastrar
  `verify-remote`, `smoke:prod:*` ni `public_main_sync`.

- `tests-node/turnero-v2-gate-contract.test.js`

## Cambios aplicados

- `AdminDataController` acepta `release.mode=web_pilot` como salida local
  válida y remapea operador, kiosco y sala al carril browser cuando el piloto
  local no exige shells nativas.
- El readiness del admin y del hardware deja desktop/Android como diferidos
  cuando la clínica está en `web_pilot`.
- `package.json` promueve `gate:turnero -> gate:turnero:web-pilot` y hace que
  `test:turnero:web-pilot:ui` use el wrapper canónico
  `bin/run-turnero-web-pilot-ui.js`.
- `bin/run-playwright-local.js` deja `stdout/stderr` del servidor PHP local en
  `ignore`, evitando el stall por buffers llenos bajo carga Playwright.
- La batería Node cubre explícitamente `web_pilot` con nativas no bloqueantes y
  mantiene `suite_v2` como carril legado soportado.

# AG-203 - Prod ops public sync telemetry adoption

Fecha: 2026-03-11

## Objetivo

Adoptar la telemetria disponible en `checks.publicSync` dentro de los
entrypoints operativos de produccion para que el diagnostico de
`public_main_sync` no se quede solo en `healthy=false` y exponga tambien
`state`, `lastErrorMessage`, `currentHead`, `remoteHead`,
`dirtyPathsCount` y `dirtyPathsSample`.

## Cambios

- `scripts/ops/prod/REPORTE-SEMANAL-PRODUCCION.ps1`
    - parsea `checks.publicSync` desde el health snapshot
    - agrega warnings `public_sync_*` para `unconfigured`, `unhealthy`,
      `age_alta` y `working_tree_dirty`
    - publica resumen `public_sync_*` en stdout para triage rapido
- `bin/powershell/Common.Warnings.ps1`
    - clasifica `public_sync_*` con impacto `observability`
    - apunta al runbook canonico `docs/PUBLIC_MAIN_UPDATE_RUNBOOK.md`
    - expone el bloque markdown/json `Public Sync`
- `scripts/ops/prod/SMOKE-PRODUCCION.ps1`
    - muestra `state`, `lastErrorMessage`, `currentHead`, `remoteHead`,
      `dirtyPathsCount` y `dirtyPathsSample` cuando se usa
      `-RequireCronReady`
    - enriquece los mensajes `[FAIL]` de `healthy` y `ageSeconds`
- `scripts/ops/prod/VERIFICAR-DESPLIEGUE.ps1`
    - propaga la misma telemetria a la salida de consola
    - enriquece `health-public-sync-healthy` con `state:lastErrorMessage`
    - agrega el asset `health-public-sync-working-tree-dirty`
- `scripts/ops/prod/README.md`
    - documenta que `REPORTE-SEMANAL-PRODUCCION.ps1`,
      `SMOKE-PRODUCCION.ps1` y `VERIFICAR-DESPLIEGUE.ps1` consumen
      `checks.publicSync`
- `tests-node/weekly-report-script-contract.test.js`
    - extiende el contrato del reporte semanal con el bloque `public sync`
- `tests-node/prod-ops-public-sync-contract.test.js`
    - agrega contrato dedicado para smoke, verify y README

## Verificacion

Comandos:

```powershell
node --test tests-node/weekly-report-script-contract.test.js tests-node/prod-ops-public-sync-contract.test.js
```

```powershell
$files = @(
  'scripts/ops/prod/REPORTE-SEMANAL-PRODUCCION.ps1',
  'scripts/ops/prod/SMOKE-PRODUCCION.ps1',
  'scripts/ops/prod/VERIFICAR-DESPLIEGUE.ps1',
  'bin/powershell/Common.Warnings.ps1'
)
$hadErrors = $false
foreach ($file in $files) {
  $tokens = $null
  $errors = $null
  [void][System.Management.Automation.Language.Parser]::ParseFile((Resolve-Path $file), [ref]$tokens, [ref]$errors)
  if ($errors.Count -gt 0) {
    $hadErrors = $true
    Write-Output ('PARSE_FAIL ' + $file)
    $errors | ForEach-Object { Write-Output $_.Message }
  } else {
    Write-Output ('PARSE_OK ' + $file)
  }
}
if ($hadErrors) { exit 1 }
```

Resultados:

- `10/10` tests Node en verde
- `PARSE_OK` para:
    - `scripts/ops/prod/REPORTE-SEMANAL-PRODUCCION.ps1`
    - `scripts/ops/prod/SMOKE-PRODUCCION.ps1`
    - `scripts/ops/prod/VERIFICAR-DESPLIEGUE.ps1`
    - `bin/powershell/Common.Warnings.ps1`

## Conclusion

La adopcion repo-side de telemetria para `publicSync` queda cerrada. El warning
que sigue abierto ya no es deuda local de scripts: corresponde al estado live
del host y requiere rollout/cron para que `public_main_sync` deje de fallar.

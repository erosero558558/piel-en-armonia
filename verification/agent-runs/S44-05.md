## S44-05 - Clinic profile API

### Resumen

- Se abrió `GET /api.php?resource=clinic-profile` como snapshot público para portal.
- El payload ahora expone nombre, logo, colores, horarios, doctores activos y servicios.
- `portal-shell` consume ese snapshot para hidratar branding dinámico en el portal.
- `portal-dashboard` usa el snapshot como fallback para ubicación, doctor y servicio en la próxima cita.

- `controllers/ClinicProfileController.php`
- `lib/ClinicProfileStore.php`

Smoke HTTP real:

```bash
python3 - <<'PY'
import os, subprocess, tempfile, time, json, urllib.request
cwd = os.getcwd()
tmp = tempfile.mkdtemp(prefix='clinic-profile-http-')
env = {**os.environ, 'PIELARMONIA_DATA_DIR': tmp, 'PIELARMONIA_SKIP_ENV_FILE': 'true'}
server = subprocess.Popen(['php', '-S', '127.0.0.1:8027', 'bin/local-stage-router.php'], cwd=cwd, env=env, stdout=subprocess.DEVNULL, stderr=subprocess.DEVNULL)
try:
    time.sleep(1.2)
    with urllib.request.urlopen('http://127.0.0.1:8027/api.php?resource=clinic-profile', timeout=5) as res:
        body = json.loads(res.read().decode('utf-8'))
        print(json.dumps({'status': res.status, 'ok': body.get('ok'), 'clinicName': (body.get('data') or {}).get('clinicName')}, ensure_ascii=False))
finally:
    server.terminate()
PY
```

```bash
const { chromium } = require('@playwright/test');
const { spawn } = require('child_process');

(async () => {

    console.log(await page.locator('[data-portal-clinic-name]').textContent());
```

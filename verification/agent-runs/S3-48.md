# S3-48

## Resumen

`bin/stuck.js` ahora registra blockers activos en `BLOCKERS.md`, libera el
claim asociado y crea el commit automático del bloqueo o de su resolución.

## Cambios

- Se refactorizó `bin/stuck.js` para soportar `BLOCKERS.md` auto-generado con
  una sección estable de blockers activos.
- El flujo `stuck` ahora guarda tarea, razón, fecha y agente en
  `data/claims/stuck.json` y en `BLOCKERS.md`.
- El flujo `clear` sincroniza `BLOCKERS.md` y genera commit automático al
  resolver el bloqueo.
- Se dejó `BLOCKERS.md` con marcadores explícitos para la sección auto-generada.
- Se agregó `tests-node/stuck-cli.test.js` con integración sobre un repo
  temporal usando `git init`.

## Validacion ejecutada

```bash
node --check bin/stuck.js
node --check tests-node/stuck-cli.test.js
node --test tests-node/stuck-cli.test.js
node bin/stuck.js list
```

Resultados:

- Sintaxis Node verde para `bin/stuck.js` y la prueba nueva.
- `node --test tests-node/stuck-cli.test.js` verde (`2 passed`), verificando:
  claim liberado, escritura de `BLOCKERS.md`, auto-commit y flujo `clear`.
- `node bin/stuck.js list` confirma que el repo real no tiene blockers activos.

## Notas

- Durante el cierre había cambios ajenos activos en la superficie de
  `appointments`; se dejaron fuera del scope de `S3-48`.

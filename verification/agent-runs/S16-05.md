# S16-05 — Contrato público de monitoring-config

## Resumen

- Se formalizó la allowlist pública en `controllers/MonitoringConfigController.php` para exponer solo `sentry_dsn_frontend`, `ga_measurement_id` y `clarity_id`.
- Se corrigió la resolución de DSN público para aceptar `AURORADERM_SENTRY_DSN_PUBLIC` y mantener compatibilidad con aliases legacy.
- Se añadió `tests/Integration/MonitoringConfigControllerTest.php` para congelar la ausencia de claves sensibles aunque existan en el entorno.
- También se corrigió `api.php`, que no estaba cargando `MonitoringConfigController.php`, y por eso la ruta pública `resource=monitoring-config` devolvía `500`.

## Validación

- `php -l controllers/MonitoringConfigController.php`
- `php -l tests/Integration/MonitoringConfigControllerTest.php`
- `php -l api.php`
- `php vendor/bin/phpunit tests/Integration/MonitoringConfigControllerTest.php`
- smoke local:
  `curl /api.php?resource=monitoring-config` → `HTTP 200`, sin `sentry_auth_token` ni `AURORADERM_SENTRY_DSN`
- `git diff --check`

## Resultado

- El endpoint público queda limitado a las tres claves esperadas y la ruta real vuelve a responder correctamente.

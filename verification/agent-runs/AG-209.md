# AG-209 - Create self-hosted fallback runner incident in repair-git-sync

## Summary

- Added a dedicated incident in `repair-git-sync.yml` for the case where the self-hosted Windows fallback is dispatched but remains `queued` or `dispatched_not_observed`.
- Kept the new incident separate from telemedicine and generic repair failures so runner capacity issues are visible as their own prod signal.
- Updated the public main update runbook to document the new incident lifecycle.

## Files

- `.github/workflows/repair-git-sync.yml`

- `docs/PUBLIC_MAIN_UPDATE_RUNBOOK.md`

## Verification

## Result

- Contract suite passed: `9/9`.
- `repair-git-sync.yml` now creates or updates `[ALERTA PROD] Repair git sync self-hosted fallback sin runner` when the fallback runner is not available/observable after dispatch.
- The incident closes automatically once repair no longer recommends the self-hosted path, or when the downstream fallback run is observed beyond the blocked states.

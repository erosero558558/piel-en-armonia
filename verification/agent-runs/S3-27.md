# S3-27 — Lista de espera para slots sin cupo

## Resumen

- Se agregó una waitlist pública al motor de booking para cuando una fecha quede sin horarios disponibles.
- La solicitud reutiliza los datos del mismo formulario de booking y guarda una entrada deduplicada por paciente/fecha/servicio/doctor.
- Cuando una cita se cancela o se mueve y libera el cupo previo, el sistema toma la primera coincidencia pendiente y encola un WhatsApp automático.

## Cambios principales

- Backend:
  - nuevo `BookingWaitlistService` para alta, deduplicación y notificación
  - nuevo `BookingWaitlistController` con `POST /api.php?resource=booking-waitlist`
  - hooks en `AppointmentController` para disparar aviso al liberar un slot por cancelación o reagendamiento
  - persistencia extendida en `StorageConfig` y `StorePersistence` para `booking_waitlist`
- Frontend:
  - CTA de waitlist dentro del shell de booking `v3/v5`
  - estado visual y copy contextual cuando un día queda sin cupos
  - cliente `data-engine` y `booking` actualizados para enviar la solicitud
- QA:
  - prueba de integración nueva para alta y notificación
  - harnesses PHP directos para dedupe y flujo cancelación -> waitlist -> WhatsApp

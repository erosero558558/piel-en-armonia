# S9-02 — Portal: recetas y certificados descargables

## Resultado

- Se publicó el historial real del portal del paciente en `/es/portal/historial/`.
- Cada atención ahora muestra dos documentos con estado explícito:
  - `Disponible`: descarga directa del PDF con el token de sesión actual.
  - `Pendiente`: el documento existe en borrador clínico pero todavía no está emitido.
  - `No emitido`: no hay documento generado para esa consulta.
- Se añadieron endpoints propios del portal para:
  - listar el historial autenticado por bearer del portal.
  - servir PDFs validando que el documento pertenezca al paciente autenticado.

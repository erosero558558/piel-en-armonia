# AG-199 - Turnero Operador Windows v1 hardening

- Date: `2026-03-11`
- Task title: `Turnero Operador Windows v1 hardening`

## Outcome

- Hardened the Electron shell under `src/apps/turnero-desktop` for the Windows operator slice:
    - `scripts/dev.mjs` now fails fast with a clear message when `electron` is missing.
    - `scripts/build.mjs` now uses dynamic imports, explains missing `electron-builder`, and relaxes Windows-only signing checks for local operator packaging (`signAndEditExecutable: false`, `verifyUpdateCodeSignature: false`).
    - `src/main.mjs` now exposes richer runtime metadata in the desktop snapshot so renderer/web surfaces can identify surface, channel, packaging mode, platform, and shell version without hardcoded assumptions.
    - `src/renderer/boot.js` now renders surface-aware packaged copy instead of always assuming a Windows operator label.
- Extended `src/apps/queue-operator/index.js` so the operator surface reports desktop bridge metadata in readiness and heartbeat payloads, refreshes shell state on boot/visibility, and exposes a clearer settings entry point for installed Windows operator PCs.
- Updated operator Windows docs in `src/apps/turnero-desktop/README.md` and pilot notes in `docs/RUNBOOK_TURNERO_SALA.md` without touching the release runbook currently owned by another task.
- Hardened admin operations/install hub rendering in `src/apps/admin-v3/shared/modules/queue/render/section/install-hub.js`:
    - install telemetry now renders multiple operator instances under one surface when two Windows PCs report heartbeats at once;
    - station routing for C1/C2 now prefers instance-level matches and falls back safely to legacy `latest` telemetry;
    - legacy `latest` heartbeats inherit live group status so operations cards keep working with old payloads;
    - consultorio board, dispatch deck, wait radar, and incident telemetry now show station-aware operator labels plus Windows shell metadata.
- Added CSS support for operator instance subcards in `queue-ops.css`.
- Added/updated Playwright coverage:
    - `tests/queue-operator.spec.js` now verifies desktop bridge metadata on the operator surface.
    - `tests/admin-queue.spec.js` now verifies two Windows operator PCs (`C1 fijo` and `C2 fijo`) appear correctly in operations and incidents, while preserving legacy single-operator behavior.
- Rebuilt the admin bundle after source edits:
    - `admin.js`
    - `js/admin-chunks/index-DxEa1U1o.js`

# S15-05 — Alertas de claim abandonado

## Resumen

- `bin/claim.js` ahora acepta `list` como alias de `status` y muestra una seccion `⚠️ Claims expiradas:` cuando existen claims vencidas.
- `agent-orchestrator.js status --json` publica un campo `expiryWarning` con el detalle de claims expiradas para consumo de dashboards.
- Se agrego soporte `AURORA_DERM_ROOT` para probar ambos flujos contra roots temporales sin depender del board real.

## Archivos

- `bin/claim.js`
- `agent-orchestrator.js`
- `tests-node/claim-cli.test.js`
- `tests-node/agent-orchestrator-stub.test.js`

## Validacion

- `node --check bin/claim.js`
- `node --check agent-orchestrator.js`
- `node --test tests-node/claim-cli.test.js tests-node/agent-orchestrator-stub.test.js tests-node/status-cli.test.js`
- `git diff --check`

## Resultado

- `node bin/claim.js list` puede advertir claims expiradas con tarea y agente.
- `node agent-orchestrator.js status` expone `expiryWarning` como `null` o como objeto estructurado cuando hay expiradas.

# S16-02 — Psalm gate por archivos cambiados

## Resumen

- Se endureció el job `psalm` en `.github/workflows/ci.yml` para que, cuando el diff toque superficies sensibles, falle explícitamente si falta `vendor/bin/psalm`.
- El mensaje operativo quedó fijado como `Instala Psalm: composer require --dev vimeo/psalm`.
- Se añadió una prueba de contrato en `tests-node/ci-psalm-workflow-contract.test.js` para fijar el trigger por `controllers/**` y `lib/**`, el guardrail del binario y la condición de no-skip silencioso.

## Validación

- `node --check tests-node/ci-psalm-workflow-contract.test.js`
- `node --test tests-node/ci-psalm-workflow-contract.test.js`
- `git diff --check`

## Resultado

- Cambiar cualquier `.php` bajo `lib/**` o `controllers/**` mantiene visible el job `psalm` en CI.
- Si Composer no dejó `vendor/bin/psalm`, el workflow falla con instrucción accionable en vez de quedar en silencio.

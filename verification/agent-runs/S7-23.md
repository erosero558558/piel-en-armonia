# S7-23 — `npm run audit` wrapper

Fecha: 2026-03-30

## Resumen

- `npm run audit` ahora apunta al wrapper compartido `bin/audit.js`.
- El wrapper ejecuta en orden exacto:
  - `node bin/velocity.js`
  - `node bin/verify.js`
  - `node bin/conflict.js --json`
  - `php -l lib/email.php`
  - `php -l controllers/OpenclawController.php`
- `gov:audit` mantiene el mismo wrapper y `gov:audit:json` conserva salida estructurada.
- Se añadió prueba de contrato en `tests-node/audit-cli.test.js`.

## Validación

Comandos ejecutados:

```bash
node --check bin/audit.js
node --check tests-node/audit-cli.test.js
node --test tests-node/audit-cli.test.js
npm run audit
npm run gov:audit:json
```

Resultados:

- `node --test tests-node/audit-cli.test.js` pasó: 3/3 tests.
- `npm run audit` ejecutó la secuencia correcta y salió con código `1` porque falló `node bin/verify.js`.
- `npm run gov:audit:json` devolvió `4/5 passed, 1 failed`, con fallo localizado en el paso `verify`.
- Los pasos `velocity`, `conflict`, `php -l lib/email.php` y `php -l controllers/OpenclawController.php` quedaron en verde.

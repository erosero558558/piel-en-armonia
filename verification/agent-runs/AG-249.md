# AG-249 - Public sync verification: resumen seguro en health publico

## Resumen

Se corrigio el contrato entre `public_main_sync` y `jobs verify`: el
`health_url` publico ahora expone `checks.publicSync` con un resumen seguro,
suficiente para que el orquestador pueda verificar el job sin depender de
`health-diagnostics` ni caer automaticamente en `registry_only`.

## Cambios

- `controllers/HealthController.php`
  - `publicPayload()` ahora incluye `checks.publicSync`
  - se publica solo un resumen seguro
  - no se exponen `repoPath`, `statusPath`, `logPath`, `lockFile` ni
    `dirtyPaths`
- `tests-node/agent-jobs-cli.test.js`
  - cobertura end-to-end para `jobs verify` consumiendo un `health_url`
    publico con `checks.publicSync`
- `tests-node/prod-ops-public-sync-contract.test.js`
  - contrato PHP para asegurar que `health` publico expone `checks.publicSync`
    y omite rutas internas

## Validacion

```bash
node --test tests-node/agent-jobs-cli.test.js
node --test tests-node/prod-ops-public-sync-contract.test.js
npm run agent:test
php -r 'define("TESTING_ENV", true); require "api-lib.php"; require "controllers/HealthController.php"; try { HealthController::check(["store" => read_store(), "requestStartedAt" => microtime(true), "method" => "GET", "resource" => "health", "diagnosticsAuthorized" => false]); } catch (TestingExitException $e) { echo json_encode($e->payload); }'
node agent-orchestrator.js jobs verify public_main_sync --json
```

## Resultado observado

- `node --test tests-node/agent-jobs-cli.test.js`
  - `3/3` OK
- `node --test tests-node/prod-ops-public-sync-contract.test.js`
  - `11/11` OK
- `npm run agent:test`
  - `508/508` OK
- `php ... HealthController::check(...)`
  - `health` publico ya devuelve `checks.publicSync`
  - el resumen incluye `jobId`, `healthy`, `state`, `failureReason`,
    `headDrift`, `telemetryGap`, `dirtyPathsCount` y `dirtyPathsSample`
  - el resumen no expone rutas internas ni el listado completo de `dirtyPaths`
- `node agent-orchestrator.js jobs verify public_main_sync --json`
  - sigue `registry_only/unverified` en produccion
  - eso confirma que el fix aun no esta desplegado en el host

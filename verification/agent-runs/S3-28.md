# S3-28 — Vista de agenda diaria en admin

## Resumen

- Se agrego un rail de agenda diaria en admin con pacientes del dia, hora, tipo, estado y alertas de overbooking.
- Se habilito la accion `Marcar llego` desde la vista operativa y desde la agenda diaria.
- Marcar llegada ahora actualiza la cita a `arrived` y avanza el patient journey clinico asociado.

## Cambios principales

- UI admin:
  - nuevo panel lateral de agenda diaria y alertas de sobrecupo
  - nueva accion `mark-arrived` en la superficie de citas
  - soporte visual para estado `arrived`
- Read model:
  - agregados `dailyAgenda`, `overbookingAlerts`, `confirmedTodayCount` y `arrivedCount`
- Backend:
  - `map_appointment_status()` acepta `arrived`
  - `PatientCaseService` refleja `arrived` en el caso clinico y persiste `summary.milestones.arrivedAt`
- QA:
  - spec nueva para agenda diaria y accion de llegada
  - ajuste de spec responsive para el nuevo CTA
  - cobertura PHP para mapping y avance de journey

# AG-248 - Focus trim: contratos para runtime checks por surface

## Resumen

Se cubrio el nuevo contrato de `required_checks` runtime por surface en la suite
del orquestador para evitar regresiones cuando el foco usa un target especifico
como `runtime:operator_auth`.

## Cambios

- `tests-node/agent-orchestrator-cli.test.js`
    - cobertura CLI para `focus status` con `runtime:operator_auth`
- `tests-node/orchestrator/status-engine.test.js`
    - render de `required_checks` por surface sin asumir rojo provider-wide
- `tests-node/orchestrator/diagnostics.test.js`
    - warnings del foco preservan el target runtime por surface cuando no hay
      snapshot runtime cargado

## Validacion

```bash
node --test tests-node/agent-orchestrator-cli.test.js tests-node/orchestrator/status-engine.test.js tests-node/orchestrator/diagnostics.test.js
npm run agent:test
```

## Resultado observado

- suites tocadas: `78/78` OK
- suite amplia: `506/506` OK
- el contrato cubre dos comportamientos distintos:
    - `focus status` puede resolver `runtime:operator_auth=green`
    - `status/diagnostics` sin verificacion runtime en vivo mantienen
      `runtime:operator_auth=unverified` de forma explicita y estable

## Nota

Durante la validacion amplia aparecio un chunk admin huerfano
(`index-LMCMM7H1.js`). Se limpio con `npm run chunks:admin:prune` y la suite
volvio a verde sin cambios adicionales de comportamiento.

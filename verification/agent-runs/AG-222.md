## AG-222 - Extraer mock OpenClaw operador a helper compartido

### Resumen

- Se agregaron `buildOperatorQueueTicket` y `buildOperatorQueueState` a `tests/helpers/admin-auth-mocks.js` para que los specs del operador reutilicen el mismo fixture canonico de cola.
- `tests/queue-operator.spec.js` ahora reutiliza `installOperatorOpenClawAuthMock` y los builders compartidos en el flujo OpenClaw del operador.
- `tests/operator-auth-shared-session.spec.js` reutiliza el mismo helper de auth OpenClaw y los mismos builders de cola para la sesion compartida entre admin y operador.
- Se extendio `tests-node/admin-auth-mocks-helper.test.js` para fijar por contrato los builders y el uso compartido del helper.

### Verificacion

- `node --test tests-node/admin-auth-mocks-helper.test.js`
    - Resultado: `7/7` en verde.
- `npx playwright test tests/queue-operator.spec.js --workers=1`
    - Resultado: `16/16` en verde.
- `npx playwright test tests/operator-auth-shared-session.spec.js --workers=1`
    - Resultado: `2/2` en verde.

### Resultado

- El flujo OpenClaw del operador y la sesion compartida `admin + operador` ya reutilizan el mismo helper de auth y el mismo fixture de cola.
- Queda fijado por contrato Node que ambos specs sigan consumiendo el helper compartido y no vuelvan a recrear utilidades locales.

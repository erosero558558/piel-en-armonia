# S3-45

## Resumen

Se extrajo el mapa de checks específicos por tarea a `bin/lib/gate-checks.js`
y `bin/gate.js` ahora imprime evidencia concreta por check en lugar de depender
solo de booleanos o mensajes genéricos.

## Cambios

- Se creó `bin/lib/gate-checks.js` como fuente compartida de checks por tarea.
- Se movieron los checks existentes del gate a ese mapa y se reforzaron los
  ejemplos pedidos para `S3-19`, `S3-24` y `S3-36`.
- Se agregó un check del propio `S3-45` para validar que el runtime del gate y
  la prueba Node usan el mapa compartido.
- Se actualizó `bin/gate.js` para aceptar resultados con evidencia y renderizar
  detalles en `PASS/FAIL`.
- Se añadió `tests-node/gate-task-checks.test.js` para cubrir los checks
  sensibles y la conexión del mapa compartido.

## Validacion ejecutada

```bash
node --check bin/gate.js
node --check bin/lib/gate-checks.js
node --check tests-node/gate-task-checks.test.js
node --test tests-node/gate-task-checks.test.js
node bin/gate.js S3-19
node bin/gate.js S3-24
node bin/gate.js S3-36
node bin/gate.js S3-45
```

Resultados:

- Sintaxis Node verde en `bin/gate.js`, `bin/lib/gate-checks.js` y la prueba nueva.
- `node --test tests-node/gate-task-checks.test.js` verde (`4 passed`).
- `node bin/gate.js S3-19`, `S3-24`, `S3-36` y `S3-45` salen en `GATE PASSED`
  mostrando evidencia explícita de artefactos y wiring.

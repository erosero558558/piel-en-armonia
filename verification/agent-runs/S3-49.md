# S3-49

## Resumen

Se agregó `bin/status.js` y el script `npm run status` para resumir en una sola
ejecución el estado operativo del repo: progreso, claims activos, ramas listas
para merge, velocidad actual y próxima fecha de revisión.

## Cambios

- Se creó `bin/status.js` como wrapper de `report.js`, `claim.js status`,
  `velocity.js --json` y `merge-ready.js --json`.
- El comando soporta salida legible y `--json` para tooling.
- Se añadió `status` a `package.json` para exponer `npm run status`.
- Se agregó `tests-node/status-cli.test.js` para validar parsing y formato del
  resumen agregado.

## Validacion ejecutada

```bash
node --check bin/status.js
node --check tests-node/status-cli.test.js
node --test tests-node/status-cli.test.js
node bin/status.js --json
npm run status
```

Resultados:

- Sintaxis Node verde en `bin/status.js` y la prueba nueva.
- `node --test tests-node/status-cli.test.js` verde (`2 passed`).
- `node bin/status.js --json` entrega payload agregado con resumen, claims,
  merge-ready, velocity y raw outputs.
- `npm run status` imprime el resumen rápido más el reporte consolidado.

# S15-04 — Generic evidence gate for explicit file refs

## Summary

- Added a generic explicit-file evidence check to `bin/gate.js`.
- The gate now scans the task line for quoted or backticked file paths like `es/paquetes/index.html` or `js/aurora-toast.js` and fails if any referenced file is missing.
- Added `AURORA_DERM_ROOT` support so the gate can be exercised safely against temporary repos in contract tests.
- Added CLI coverage in `tests-node/gate-cli.test.js` for both the failing and passing cases of the generic evidence check.

## Validation

- `node --check bin/gate.js`
- `node --test tests-node/gate-cli.test.js`
- `node --test --test-name-pattern "S3-19|S3-36|S3-45" tests-node/gate-task-checks.test.js`
- `git diff --check`

## Result

- Tasks without a bespoke entry in `gate-checks.js` now still fail if they explicitly mention a missing artifact.
- The synthetic CLI harness proves the gate fails on a missing `js/aurora-toast.js` reference and passes once the file exists.

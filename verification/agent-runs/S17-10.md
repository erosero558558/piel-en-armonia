# S17-10 — Motor de promociones con elegibilidad

## Resumen

- Se añadió `lib/promotions/PromotionEngine.php` con reglas canónicas `first_consult`, `member_loyalty` y `referral_welcome`, elegibilidad por `primera_vez|miembro|referido`, exclusiones y vigencia.
- Se expusieron `GET /api.php?resource=active-promotions`, `GET /api.php?resource=promotion-config` y `POST /api.php?resource=promotion-config` vía `controllers/PromotionController.php`, `api.php`, `lib/routes.php` y `lib/ApiConfig.php`.
- Se preservaron `memberships` y `promotion_config` en storage con `lib/storage/StorageConfig.php` y `lib/storage/StorePersistence.php`.
- Se agregó panel admin para activar/pausar promociones en Settings y se corrigió el estado local del toggle para que no rebote al repintar.
- El booking runtime ahora consulta promociones activas y renderiza beneficios aplicables en V3/V5; la cobertura visual usa un harness publicado de booking porque la superficie pública V6 actual mantiene la reserva general en modo mantenimiento.
- Se restauró además el import faltante de `renderReviewsSection` en el template de secciones admin, necesario para que el admin vuelva a bootear y la prueba de Settings sea ejecutable.

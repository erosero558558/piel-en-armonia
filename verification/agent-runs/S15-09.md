# S15-09 — Regression watch

## Resumen

- Se creó `bin/regression-watch.js` como watchdog de regresiones sobre tareas `[x]`.
- El script extrae archivos explícitos desde `AGENTS.md`, cruza esa lista con `git diff --stat` y `git diff --name-status` de la ventana reciente, y alerta únicamente si el archivo mencionado quedó eliminado o en 0 bytes.
- La salida soporta modo texto y `--json`, lo que deja la base lista para conectar el reporte diario después.

## Archivos

- `bin/regression-watch.js`
- `tests-node/regression-watch-cli.test.js`

## Validación

- `node --check bin/regression-watch.js`
- `node --test tests-node/regression-watch-cli.test.js`
- `node bin/regression-watch.js`
- `git diff --check`

## Resultado

- El watchdog no produjo falsos positivos en el repo actual: `node bin/regression-watch.js` cerró con `✅ 0 regresiones detectadas`.
- El contrato cubre explícitamente el caso sano y el caso de regresión por archivo borrado en un repo temporal controlado.

# AG-156 - FE-V6-R11 harden canonical V6 build runner

## Scope

- Replace the fragile shell-chained `build:public:v6` command with a dedicated Node runner.
- Preserve the canonical V6 source model and git-sync publish contract.
- Add a contract test so the build entrypoint cannot silently regress back to chained shell commands.

## Files

- `bin/build-public-v6.js`
- `bin/assert-public-v6-single-source.js`
- `package.json`
- `tests-node/public-v6-build-contract.test.js`
- `docs/public-v6-canonical-source.md`
- `docs/DEPLOYMENT.md`
- `docs/PUBLIC_MAIN_UPDATE_RUNBOOK.md`
- `verification/public-v6-canonical/artifact-drift.json`

## Implementation

1. Added `bin/build-public-v6.js` as the canonical public V6 runner.
2. Moved the sequence into Node:
   - `content:public-v6:validate`
   - `astro:build`
   - `astro:sync`
   - `check:public:v6:artifacts --skip-build`
3. Added `verification/public-v6-canonical/build-report.json` output for step-level evidence.
4. Updated `package.json` so `build:public:v6` points to the dedicated runner.
5. Added `tests-node/public-v6-build-contract.test.js` to freeze the contract.
6. Hardened `bin/assert-public-v6-single-source.js` so it fails if `build:public:v6` stops pointing to the dedicated runner.
7. Updated canonical deployment docs to describe the new build runner and report path.

## Validation

```bash
npm run build:public:v6
node bin/assert-public-v6-single-source.js
npm run test:public:v6:build:contract
npm run test:frontend:qa:v6
```

## Notes

- The practical goal was to avoid a previous failure mode where Astro built successfully but root artifacts were not synced before publish.
- The new runner now makes sync and drift verification part of the canonical build path instead of an external shell chain.

# AG-238 - Documentar y blindar el fallback clínico del dashboard admin

## Resumen

- Se fijó el contrato DOM del bloque clínico del dashboard admin para cubrir
  `clinical-history` como sección navegable y sus nodos de resumen, acciones,
  cola y feed.
- Se agregó un contrato Node para `buildClinicalHistoryActions(...)` que
  asegura el fallback útil cuando no hay sesiones clínicas visibles pero sí hay
  telemedicina o `patientFlow` pendientes.
- Se endureció la spec Playwright del dashboard para esperar el runtime
  `data-admin-ready="true"` antes de validar la acción clínica de fallback.

## Resultado

El fallback clínico del dashboard quedó cubierto en tres capas: contrato DOM,
contrato unitario de markup y spec end-to-end estable. Con esto, el panel ya no
depende solo del estado “feliz” de historias clínicas para ofrecer una acción
operativa útil hacia el frente clínico.

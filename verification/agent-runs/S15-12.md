# S15-12 — README stats sync

## Resumen

- Se creó `bin/gen-readme-stats.js` para generar un bloque parcial de `README.md` desde `AGENTS.md`.
- El bloque usa los marcadores `<!-- STATS_START -->` y `<!-- STATS_END -->` y muestra done/total/pct junto al sprint activo.
- `bin/sync-backlog.js` ahora dispara también la actualización de `README.md`, para que backlog y README se mantengan alineados en el mismo flujo de sync.

## Archivos

- `README.md`
- `bin/gen-readme-stats.js`
- `bin/sync-backlog.js`
- `package.json`
- `tests-node/gen-readme-stats.test.js`

## Validación

- `node --check bin/gen-readme-stats.js`
- `node --test tests-node/gen-readme-stats.test.js`
- `node bin/gen-readme-stats.js`
- `node bin/sync-backlog.js`
- `grep -n "Progreso real del board\\|Sprint activo" README.md`
- `git diff --check`

## Resultado

- `README.md` ya refleja métricas reales del board y deja de depender de números hardcodeados.
- El sync diario del backlog ahora arrastra también la actualización de ese bloque del README.

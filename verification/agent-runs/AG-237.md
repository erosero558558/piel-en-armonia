# AG-237 - Consolidar mocks del piloto web en admin queue spec

## Resumen

- Se consolidó el clúster repetido de mocks del piloto web dentro de
  `tests/admin-queue.spec.js`.
- El spec ahora usa builders locales para `turneroClinicProfile`,
  `queueSurfaceStatus`, `health` y el payload `data` del piloto, en lugar de
  repetir routers `api.php` casi idénticos en cada caso.
- Se migraron los casos de reset de clínica, drift de ruta, fallback local,
  drift de `clinic_id`, firma desactualizada, mismatch de `/health`,
  fallback por `clinic-profile` faltante y bloqueo operativo del admin.

## Archivos

- `tests/admin-queue.spec.js`

## Cambios

- Se agregó `buildQueueIdleState(...)` para el estado vacío canónico del
  turnero.
- Se agregó `buildQueuePilotClinicProfile(...)` para armar perfiles clínicos del
  piloto con overrides acotados.
- Se agregó `buildQueuePilotSurfaceStatusEntry(...)` y
  `buildQueuePilotSurfaceStatus(...)` para unificar el shape de `operator`,
  `kiosk` y `display`.
- Se agregó `buildQueuePilotHealthPayload(...)` para centralizar `publicSync` y
  checks adicionales como `turneroPilot`.
- `installAdminQueueApiMocks(...)` ahora acepta `dataOverrides`, lo que permite
  reusar el mock base sin rehacer el `resource=data` en cada test.
- Se agregó `installQueuePilotApiMocks(...)` como wrapper local del spec para
  los casos del piloto web.
- Se migraron estos casos a los helpers nuevos:
    - `queue reinicia el estado local del piloto cuando cambia de clínica`
    - `queue bloquea el piloto web si una superficie reporta otra ruta canónica`
    - `queue bloquea el piloto web si el perfil clínico solo existe en fallback local`
    - `queue bloquea el piloto web si una superficie reporta clinic_id de otra clínica`
    - `queue bloquea el piloto web si una superficie trae una firma de perfil desactualizada`
    - `queue advierte en readiness si /health reporta otra clínica activa para el piloto`
    - `queue bloquea el piloto web si una superficie usa perfil de respaldo por clinic-profile faltante`
    - `queue bloquea acciones operativas del admin si admin.html#queue queda fuera del canon del piloto`

## Resultado

El bloque del piloto web quedó mucho más mantenible sin cambiar el contrato del
hub de `queue`: el spec conserva la cobertura funcional, pero ahora los casos
del piloto comparten un vocabulario único para perfil, superficies y readiness.
